§1 · 背景

问题设定（30 秒）

让大语言模型 自动完成 DFT 计算：给定一道题（材料 + 目标物理量），agent 自行编写 Quantum ESPRESSO 输入、运行 pw.x、解析输出、自检并交卷。一个独立 bench 对照标准答案自动打分（0–1），故"能力强弱"是可量化、可对照的标量。
任务类型
  • T1 · vc_relax（结构弛豫）
  • T2 · bands_dos（能带 + 态密度）
里程碑（难度阶梯）
  • M1 元素 / 简单二元
  • M2 复杂氧化物
  • M3 磁性材料（自旋极化）
本报告范围
  • 三 solver × 6 个 (里程碑×类型) 格
  • 共同卷 ~43 题
  • 快照 + 交接，不判输赢

注：当前实验未跑全，主表为 scaffold——已有数据已填、待补格留空，末节（§8）专列下一步实验。

§6 · 口径与已知坑（读数前必看）

威胁有效性 · 与 bench 打分口径

混 build（本快照主要限制）
v5.1 标准核来自 ac19（落后 7 提交）、部分 M3 来自 6efa（磁性修复前）——表头记 ee050a2 但若干格是旧 build。每格已标 build；最干净做法 = 在冻结点重跑 v5.1（§8·P3）。
v5.1 的 0.683 ≠ 冻结点
那是 6efa15e 的成绩；ee050a2 增补磁性恢复 fbd53da，全集尚未完整测过。勿将 0.683 当作 ee050a2 成绩引用。
bench 打分口径 4 条 · 待 DFT 专家确认（非 solver 之过）
  • k-label 非命名点：VBM/CBM 落在非高对称点 → 丢满分（MnO/NiO 实测）
  • 绝缘体 dos_e_fermi 参考系：绝对能量 ±0.1 eV 比 → 系统偏移直接判 0（疑致 Fe₂O₃/Cr₂O₃-T2 三版皆低）
  • 磁矩绝对 0.5 µB 容差：对大磁矩偏松、小磁矩极松，占 T1 权重 0.35
  • null GT：Li₃PO₄ 的 k-label 全 null，null-vs-null 不给分 → 0.2 权重无法获得
按改动边界：只报告、不自行改 bench。

另：Ni_fcc 自旋通道抽签致高方差 → 金属格 best-of-3 缓解。收数时 1 个跨 user run 目录不可读（权限），已跳过。

§7 · 交接

现状与未竟事项

各版本一句话
  • v4.2 通用成熟，标准核稳；M3 非其主场
  • v4.3 磁性单题质量最高；短板为大氧化物易超时（8/12）
  • v5.1 工程更优、鲁棒性反超（12/12）；金属能带兜底开发中
未竟事项 / 风险
  • v5.1 工作树有未提交 metal_bands_guarantee.py（自旋通道兜底）——冻结于 ee050a2，不参测
  • deploy 一致性：config.toml 的 solver_path 已指 /srv 权威源（已核）
  • 批跑工装：run_subset42_batch.py 仍硬编码迁移前路径 → 改用薄 harness/run.py 驱动
  • 两条线暂不判输赢：各有所长，均在迭代
改动边界
本轮只动实验工装（manifest / 驱动 / 收数 / 报告），不改任何 solver 代码；bench 侧问题只报告、不自行修改。
