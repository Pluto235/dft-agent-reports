Solver v3 + DeepSeek-v4-pro · 12 题快速档 vs baseline

本报告把用户新版 Solver v3（Web UI 快速档）+ DeepSeek-v4-pro跑的 12 道题（4 T1 + 8 T2）跟 baseline-2026-05-10（同型号 deepseek-v4-pro，warpagent-qe-v1 solver，GPU backend）做同题对照。两边模型完全一致，差异主要来自 solver 版本（v1→v3）与硬件后端（GPU→CPU）。
测试设置：用 deepseek-v4-pro 模型 + 本仓库 v1 solver （warpagent-qe-v1），在 questions_subset42 子集上跑 21 道 T1。
硬件：CPU only（qe-cpu/q-e/bin/pw.x）。 报告生成时间：2026 年 05 月 12 日 02:20 UTC。

Agent Description (paper §3 style)

下面这段按论文 §3 v1 minimum baseline 的同款笔触描述本次 v3 solver 的设计，可直接放进 paper 的对应小节。原文为英文以便复制。

Engineered Agent (v3)

The v3 agent inverts the v1 baseline’s “smallest tool set” assumption: instead of adding tools, it concentrates domain knowledge and procedural scaffolding into static system context and a Python helper module, leaving the tool surface unchanged. The agent exposes the same four primitives as v1 — bash(cmd, timeout_s), write_file(path, content), read_file(path, offset, limit), and submit(task_outputs, notes?) — but each tool call is preceded by a markedly richer briefing.

Three components are added around the loop. First, a domain-specific system prompt (377 lines, against 42 in v1) enumerates QE-specific conventions: per-task-type input templates for vc_relax and the SCF → NSCFbands → bands.x → NSCFdos → dos.x bands_dos pipeline, recommended ecutwfc / ecutrho ranges, k-mesh defaults that depend on whether the target is a semiconductor or a metal, the convergence triple (press_conv_thr, forc_conv_thr, etot_conv_thr) the audit will check, the GBRV pseudopotential path resolution, and a CPU -np N recommendation table indexed by core count. Second, a Python helper module qe_tools.py ships with the solver and is importable from within any bash call via a python3 heredoc; it provides write_pw_input, write_bands_input, write_dos_input, and parse_pw_output, so the LLM can construct and parse QE artefacts without re-deriving the namelist grammar each turn. Third, a pre-spawn lint hook intercepts every bash command that invokes pw.x, parses the referenced input file, and auto-tightens any vc_relax convergence thresholds looser than the audit-required defaults; the patches surface back to the model as a --- qe_input_lint --- block inside the tool result, so the next turn knows the file was edited.

The loop itself remains an unstructured ReAct cycle, but the system prompt instructs the model to follow an explicit five-phase decomposition — Smoke (pw.x --version), Plan, Execute, Verify (grep "JOB DONE." plus parse_pw_output(…).convergence), and Submit — and the solver mirrors the bench audit (file ≥ 1024 B, “JOB DONE.” marker in last 64 KB, paths inside workdir) before writing result.json, so audit-fatal mistakes surface as agent-side incomplete reasons rather than post-hoc zero scores. Prompt caching with three cache breakpoints (system / tools / user) is enabled by default, exploiting the fact that the static portion of the prompt is large; on this 12-task subset the measured cache-hit rate is 97.4 %. Resource constraints follow the paper-default configuration max_turns = 100, max_output_tokens = 16384, and solver_timeout = 7200 s, identical to v1.

Compared to v1 on these twelve same-task pairs, v3 keeps the tool surface and budgets unchanged but consumes 33 % less API cost ($0.32 vs. $0.48), spends 19 % less AI wall-clock time (2.06 h vs. 2.54 h), and lifts the pass-rate at score ≥ 0.5 from 5 / 12 to 6 / 12. The largest quality gains are on materials where v1 mis-specified namelists or k-meshes — HfO2 monoclinic 0.32 → 0.604, Cd 0.45 → 0.965, and α-SiO2 0.00 → 0.25 — consistent with the hypothesis that codified domain context closes the gap on tasks that fail in v1 because of QE-grammar errors rather than because of insufficient reasoning. The remaining two failures (GaAs vc_relax and Rb bands_dos) are not v1-style long-tail failure modes: in both cases the model produced valid QE inputs, ran the pipeline at least once (Rb in fact wrote complete bands.dat and dos.dat files), then chose to retry — on a symmetry mismatch (GaAs) or with spin-polarisation enabled (Rb) — and exhausted the 2 h wall-clock budget mid-retry without invoking submit. This points to a component the present design does not address: budget-aware termination that forces a best-available submission when wall-clock margin falls below a threshold, rather than indefinite local optimisation.

3.1 失败原因复盘

本批次有 2 道题没拿到 result.json。各自的失败模式如下：

  • M1_T1_GaAs_vc_relax (花掉 $0.0285, AI 661s, 0 轮)：vc_relax 反复重试耗尽 2h 预算。早期一次 pw.x 触发 Error in routine checkallsym（晶系/坐标对称性错配），agent 多次调整 celldm(1)、k-mesh（6×6×6→4×4×4）重跑，其中一次 SCF 跑了 60min 仍未收敛；最后一次 pw.x 启动后被 harness 2h 上限 reap，未提交 result.json。
  • M1_T2_Rb_bands_dos (花掉 $0.0404, AI 969s, 0 轮)：完成 SCF + bands NSCF + bands.x + DOS NSCF + dos.x（scf.out 有 JOB DONE，bands.dat / dos.dat 已产出），但 agent 判断结果不达标，启动 spin-polarized 全流程重跑（scf_sp.in→nscf_sp.in→bands_sp.in→再准备 bands_sp2.in），在构造下一份输入时被 2h 上限 reap，未提交 result.json。

共性：都是 agent 把 2h 预算 （solver_timeout_sec=7200）耗在 QE 重试 / 流程重跑上，没有"提交一份 不完美的答案"的兜底机制。一种可改的方向：在 timeout 还剩 ~10% 时强制让 agent 封装现有最好结果提交，至少能拿到部分分。
