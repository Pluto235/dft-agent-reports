1 · 起因：上一版 solver 老在哪些 DFT 细节上栽跟头

我们最早的版本（叫 v1）很简单：一个 LLM，给它三个工具 （bash 跑命令、write_file / read_file 读写文件）， 再丢一段几十行的"你是 QE 用户"的系统提示，让它读 bench 给的题目（task prompt）、 自己想、自己写 QE 输入、自己跑、自己判断收敛没收敛、自己交答案。形式上就是"边想 边干，干完看一眼，再想再干"。

v1 像你临时雇了一个远程同事，他通读过 QE 文档但没真做过项目。你把题目甩过去， 他会试着做；但你不在旁边，他怎么挑参数、查没查输出、什么时候放弃，全凭他自觉。

在 6 道最简单的题目（Si / NaCl / Cu 这类单元胞 vc-relax）上 v1 能拿到 0.7+ 的均分。 但稍微难一点，就会暴露这几类系统性问题——熟悉 QE 的同行一看就知道是怎么回事：

v1 上经常踩的坑对应的物理 / bench 后果
vc-relax 用了 QE 的默认收敛阈值 （press_conv_thr=0.5 kbar、forc_conv_thr=1e-3 Ry/Bohr、 etot_conv_thr=1e-4 Ry） cubic 系统的弛豫结构漂移 ≥0.01° 或 ≥0.05%，pymatgen 的 SpacegroupAnalyzer 把它降级为 trigonal/orthorhombic，空间群分扣 0.25。实测 ~25% miss 率。
SCF 不收敛 → 死循环 LLM 反复改 mixing_beta / 加 nstep，2 小时墙时间用完， 最后什么也没提交。
金属用了 occupations='fixed'， 或者绝缘体用了 smearing SCF 振荡 / 假收敛；smearing 把 gap 抹平，bench 评分崩。
T2 (bands+dos) 直接上来跑 bands 结构没先 vc-relax，bands 结果建立在错的几何上；或者 PBE+USPP 用了 ecutrho=4×ecutwfc（NC pseudo 的默认）→ 带隙漂 0.2–0.5 eV。
bash 命令里跑了 pip install / curl bench audit 要求纯离线运行，看见网络命令直接判 0 分。
LLM 自己说"我做完了" 漏交付字段 / 提交未收敛的 .out / 给的路径根本不存在 → bench 一查就 0 分。

把这几类坑列出来看会发现一个共性：不是 LLM 笨，而是 v1 这套架构里没有 "在它瞎来之前拦一下"的机制。LLM 写了松收敛的 vc-relax，没人事先收紧； 跑完它说"做完了"，没人独立查文件；它陷在死循环里，没人决定"该重试还是该换思路"。 v4.2 做的事，本质就是把这几个"没人做"的环节补上。

2 · v4.2 整体怎么工作的：三个 LLM 角色接力 + 两道前置功课

v4.2 不再是一个 LLM 单打独斗，而是分工流水线。bench 把一道题（比如 "算 Si 的 vc-relax 后晶格常数和空间群"）丢给 solver，solver 大致这样跑：

图 1 · v4.2 的整体流程。两道前置功课 + 三个 LLM 角色接力 + retry / replan 反馈环。  v4 另：每次 pw.x 跑完会自动解析结构化 facts（converged / energy / iter / force / stress / wall_time）写进 qe_facts.jsonl，verifier 直接读它，不再 grep .out 末尾。

三个 LLM 角色各管一段，互相不串台：

  • planner：只读题目，输出一份"分几步、每步做什么"的清单 （比如"第一步 vc-relax，第二步收集结果"）。它不读文件、不跑命令， 只决定"工作流是什么样的"。
  • executor：拿到 planner 写的某一步，去实际写 QE 输入、跑 pw.x、看输出。 每一步换一个全新的 LLM session——上一步算 vc-relax，下一步算 bands，executor 不会被上一步的细节污染。
  • verifier：用一份"这一步该长什么样"的清单 （比如 vc-relax 步骤必须有 bfgs converged in 字样、 cubic 系统 a/b/c 要在 0.01 Å 内一致）去查 executor 留下的文件。 通过就放行，不通过就给一句话原因（这句原因决定下一步怎么走）。

verifier 不通过怎么办？ 同一步先让 executor 看着 verifier 的原因重试， 最多重试 2 次。还不行就退到 planner，让它根据这个失败原因写一份新计划——这叫 "replan"， 最多 5 次。最终都救不回来才提交 incomplete。

v1 是"一个人闷头干"；v4.2 是 "项目经理（planner）写 spec， 工程师（executor）实施，QA（verifier）按清单验收，验收不过先返工再改方案"。 每个角色都可以独立换模型——目前 planner / executor 用 Claude Sonnet，verifier 用更便宜的 Haiku， 反思（见 §4.6）也用 Haiku。

整个过程跑完，agent 会写两个文件：

  • result.json 交给 bench 打分（bench 要的字段路径、文件内容）。
  • 全过程审计记录（每步谁说了什么、planner 写过哪几版计划、verifier 判过哪几次失败）。 我们用它复盘——任何一道题跑得好不好，都能精确还原。

3 · v4.2 怎么把上一版的"坑"一个个堵上

下面把 §1 那几类 v1 的坑，逐一对应到 v4.2 里是哪一个机制在挡。核心哲学： v1 把 DFT 知识"告诉" LLM 让它自己用；v4.2 把同样的知识写成代码层的强约束， LLM 想绕都绕不过去。

v1 上的坑v4.2 怎么挡
vc-relax 阈值太松，空间群退化 每次准备跑 pw.x 之前，agent 先把 LLM 写的输入文件交给 lint 检查； 发现 vc-relax 的三阈值不够紧，自动把它收紧到 press_conv_thr=0.05 / forc_conv_thr=1e-4 / etot_conv_thr=1e-5， 再 spawn pw.x。LLM 自己写多松都收紧。
SCF 死循环，跑墙时间 每步最多重试 2 次；耗尽就回去 planner 重写计划，最多 5 次。 全跪了就 submit incomplete，不让它拖死。
金属用 fixed / 绝缘体用 smearing 开题之前先做"金属还是绝缘体"的判别——化学式正则 + 元素列表 + 简单规则（含 d/f 元素 / 强电负性元素的组合）。结果作为 advisory 塞进 planner / executor 的提示词；executor 还会被自动监督 occupations 和这个建议是否一致。
USPP 用了 ecutrho=4×ecutwfc lint 看到 pseudo 家族里有 USPP / GBRV / PAW，自动把 ecutrho 改成 10×ecutwfc（GBRV 标准）。
bands 用了 calculation='bands' 模式 lint 看到 K_POINTS crystal_b + calculation='bands'， 自动改成 'nscf'——bench evaluator 要从 nscf 模式才会写的 highest occupied / lowest unoccupied 行里抓 VBM/CBM。
T2 直接 bands，没先弛豫几何 开题就先单独跑一次 vc-relax（不是 planner 计划的，是写死的前置功课）， 把松弛后的几何和能量塞进主流程的提示词。主 LLM 不再操心这一步。
bash 跑 curl / pip install bash 工具内部有黑名单：curl / wget / ssh / pip install 之类 在 spawn 之前就被拒。pw.x 后台化（& / nohup / setsid）也被拒。
LLM 自己说"做完了"，但漏字段 / 提交未收敛 verifier 接管"做完了"这个判断。每一步对应一份"这一步至少要有 XX" 的清单（见 §4.3），verifier 拿清单逐项查文件。
bench 评分时用 fermi-slice 抽 VBM/CBM，有时漏算金属 每跑完 T2 自动加一步 物理一致性检查：拿开题判定的"金属/绝缘体" 和 bench 抽出的 gap_type / band_gap_ev 比对，矛盾就 fail + replan。 例如已知 Rb 是金属但 bench 抽出 1.2 eV gap → 提示加大 nbnd + 改 k-path 覆盖 Fermi 穿越点。
agent 绕过 GPU wrapper 直接调 /home/.../pw.x，CPU 跑 2 小时还没出 vc-relax.out（WO3 案例）
v4 新增 · FIX-4		 bash 工具拒绝任何绝对路径的 QE 可执行文件（pw.x / bands.x / dos.x / ph.x / pp.x / ...）； 强制 agent 走 GPU wrapper 自动检测路径。spawn 前就拒，错误信息直接回写给 LLM 让它修 PATH 而不是换 binary。
agent 给重 step 设太短 timeout，反复自砍重启 （CoSb3 案例：dos_nscf 给 120s，实际 32 原子方钴矿 dense kmesh + tetrahedra 要 20+ min）
v4 新增 · FIX-5		 按工作流阶段强制 timeout 最低值（scf=300s / vc_relax=900s / bands=600s / dos=1800s / pp=60s / physics_sanity=60s）。 bash 工具拿到 LLM 的 timeout_s 后取 max(LLM 值, 阶段下限)—— agent 可以加长但不能再砍。
verifier 反复 grep .out 末尾找 JOB DONE / energy / 收敛行（重复成本 + 容易看错）
v4 新增 · PR-B		 每次 pw.x 跑完，bash 工具自动解析结构化 facts （converged / total_energy_ev / n_iter / max_force / max_stress / wall_time_s / error_signature）写进 qe_facts.jsonl（每行一次调用）。 verifier 直接 read 这个 cache，物理验收速度 + 准确率都上去了。
executor 每步当孤岛，不知道全局——一个 vc_relax 调到极致 但后面 bands 时间不够
v4 新增 · PR-C		 executor 启动时收到一份"我之后还要跑哪几步"清单（只显示 id+kind， 不显示 step 的 desc 防它偷跑下一步）。让 executor 有"ship correct, move on" 的全局意识。

整张表里只有"金属/绝缘体的判定结论" + v4 新增的 §4.2 材料家族 playbook 这两条走"prompt 注入靠 LLM 自觉"——其他全部是代码层强约束 （spawn 前的 lint / 拒绝 bash 黑名单 / 独立的 verifier 查清单 / 状态机决定重试还是放弃）。 这就是 v4.2 的核心改动。

4 · 我们写下来"代 agent 做主"的 6 类 DFT 决策

下面 5 个机制每一个都把具体的 DFT 物理判断写死到了代码或提示词里。 这就是这份报告想请你审的核心——这些默认值 / 判定规则，物理上对吗？漏了什么？

§4.1

写进 system prompt 的 QE 通用纪律

每个 executor 启动时拿到的提示词里，硬写了下面这些 "QE 老手都知道、文档第一页不写"对所有题目都适用的纪律。 （针对特定材料家族的额外说明书见 §4.2。） 新手 agent 第一次写 QE 输入很容易忘，所以我们直接告诉它"不准这样做、必须那样做"。

  • 赝势路径纪律：必须 pseudo_dir='./pseudos/'。 否则 pw.x 找不到赝势文件，Error in routine readpp。
  • vc-relax 三阈值（同时在 §4.5 lint 强制收紧）： press_conv_thr=0.05 / forc_conv_thr=1.0d-4 / etot_conv_thr=1.0d-5。还附了完整 vc-relax 输入模板。
  • T2 (bands+dos) 标准参数：bench 的 GT 用的是 ecutwfc=80, ecutrho=800 (10×, GBRV USPP 标准), kmesh ≈ ceil(50/a_Å), 最小 8×8×8。 评分用 ±0.1 eV 容差，参数松一点就会确定性扣分。
  • bands NSCF 一定用 calculation='nscf'， 不是 'bands'——只在 nscf 模式 QE 才输出 highest occupied, lowest unoccupied level 那行，bench 靠这行抓 VBM/CBM。
  • DOS 用 occupations='tetrahedra'；dos.x 用 deltaE=0.01 eV, ngauss=0, degauss=0.0015 Ry (=0.02 eV)—— 匹配 GT 的能量栅格和展宽。否则 DOS 形状的 RMSE 直接崩。
  • 晶格 6 标量 → CELL_PARAMETERS 必须用 pymatgen Lattice.from_parameters， 不许手算。rhombohedral 里把 a*sqrt(2/3) 错写 a*sqrt(8/3)， 体积翻倍，metal/insulator 都判错。
  • k-path 用 seekpath.get_path，不许手挑高对称点。
  • occupations 怎么选：半导体 / 绝缘体 → 'fixed'； 金属 → 'smearing'（mv 函数，degauss=0.02 Ry）。

这些不是 QE 文档第一页能看到的，都是过去 v1/v3/v4.1 至少踩过一次的失分场景。但 DFT 圈一定还有"大家都这么做但论文不写"的细节我们没注入—— 这就是 §6.1 想请你看的。

§4.2

针对特定材料家族的 playbook（v4 新增）

§4.1 写的是对所有题目都适用的 QE 通用纪律。 但 DFT 圈都知道——碱金属 / 复杂硅酸盐 / 低对称氧化物 / 极性 III-V 这几类 材料各有"不一样的玩法"，通用纪律不够用。v4.2 给每一类加一份独立的 playbook， 开题时按材料的公式 / 元素 / 晶系 / 材料族匹配后注入 planner + executor 的提示词。

当前 4 份 playbook 概览：

适用范围触发条件关键原则节选
alkali-metal-advisor
Rb / Cs / 纯碱金属		 material_family ∈ {alkali_metal}
  • 纯碱金属是金属，SCF 必须用 smearing（occupations='fixed' 不收敛）
  • smearing 函数用 mv 或 mp；degauss 取范围下限
  • k-mesh 密度比绝缘体高得多（Fermi 面需要采样）
  • 重碱（Rb / Cs）可能要 scalar-relativistic pseudo
complex-silicate-advisor
spodumene LiAlSi₂O₆ 类
（Si-O + 额外阳离子）		 material_family ∈ {silicate}
  • 必须先 vc-relax 再 production SCF（数据库几何不准）
  • 紧 forc_conv_thr（软模 / 倾转模式容易假收敛）
  • DOS 用比平常更密的 k-mesh（cation 平带需要解析）
  • 纯 Si-O 框架（quartz / cristobalite）不算 silicate—走 oxide
low-symmetry-oxide-advisor
HfO₂ baddeleyite / V₂O₅ / 单斜三斜 oxide		 material_family ∈ {oxide} ∧
crystal_system ∈ {monoclinic}
  • k-mesh 各向异性（按实空间晶格长度反比），禁止手挑 N×N×N
  • vc-relax 不准把 cell_dofree 锁到高对称（'ibrav' / '2D' 都不行）
  • 收敛比 cubic 慢（更多 SCF iter / 更多 relax step）
  • 紧 forc_conv_thr（off-diagonal 方向上的力很小）
polar-iii-v-advisor
GaN / AlN / InN / GaAs / InP / ...		 material_family ∈ {polar_III_V}
  • 不准 smearing（这是绝缘体）
  • wurtzite 必须让 c/a 独立松弛（禁止锁 c/a = √(8/3)）
  • k-path 必须覆盖 Γ / A / M / K / L（wurtzite）或 Γ / X / L（zincblende）
  • LDA/GGA 系统性低估带隙约 2 倍—题目要"DFT 带隙"还是"实验值"看清

关键设计：playbook 不准写数值

每份 playbook 文档只准写"物理原理 + 失败模式 + 反例"，不准写 任何带单位的数值（不准说 "ecutwfc=80 Ry" / "degauss=0.02 Ry" 这种）。loader 在加载时跑 audit：发现违规直接拒绝注入，写诊断到 log。

理由跟 §4.6 笔记本子系统是同一个哲学——这些 playbook 是给 agent 看的 思维框架，不是答案缓存。bench 评的是 agent 的通用 DFT 能力 而不是查表能力。如果说 "Rb 的 ecutwfc 用 60 Ry 就够"，下次跑 Rb 直接命中， 看起来"学会了"，本质是答案泄漏。

匹配 + 注入策略

开题时按 YAML frontmatter 里的 trigger 字段 declarative 匹配——可以匹配 formula / elements / crystal_system / material_family 任意组合。多份 playbook 命中按 priority 字段 排序（当前 4 份都是 priority=10 并列注入）。匹配不到任何一份 → 空 advisory 段， 不影响主流程。

这 4 份 playbook 是我们目前覆盖的 4 个 DFT 常见 family。还有哪些材料族该加？ 现有 4 份的物理判断对吗？请教 → §6.2。

§4.3

按工作流阶段做的物理验收清单

每个工作流阶段（SCF / vc-relax / bands / dos / ...）都对应一张 "这一步如果做对了，至少应该有 XX"的清单。verifier 拿这份清单去查 executor 留下的文件——通过就放行，不通过就给一句话告诉 planner 怎么改。

清单现在大多是"文件存在 + 字符串匹配"层级，真正的物理 sanity 还薄。 下表列各阶段都在查什么：

阶段必查项失败诊断举例
scf log 末尾 64 KB 含 JOB DONE.；不含 convergence NOT achieved； out/<prefix>.save/charge-density.dat 存在且非空 "out/si.save 为空 → 电荷密度没写出来"
vc_relax JOB DONE + bfgs converged in 或 End final coordinates； 无 maximum number of steps has been reached； 最终 stress ≤ 0.05 kbar；声明 cubic 的体系最终 a/b/c 在 0.01 Å 内一致 "BFGS 未收敛——bench 会按未收敛结构打分"
relax 同 vc_relax 但不查 stress / cell drift同 vc_relax
bands 所有 *_out 含 JOB DONE；bands.dat ≥1024 B 且像数据列 "bands.dat 只有 200 B → bands.x 没正确跑完"
dos nscf log 含 JOB DONE；dos.dat ≥1024 B 是数值列；无 Error in routine "dos.dat 全 0 → fildos 路径写错或 dos.x crash"
collect 纯文件操作：每个要交付的产物存在 + 非空 "目标路径没生成——是 cp 路径写错？"
physics_sanity 不调 LLM 的纯 Python 步骤。T2 任务自动加在最后。把"开题判定的金属/绝缘体" 和 bench 抽出的 gap_type / band_gap_ev 对一遍，矛盾就标 fail "已知 Rb 是金属但抽出 1.2 eV gap → 加大 nbnd 后重做 bands NSCF"

physics_sanity 这一步具体在查什么，最近一轮扩到了 6 条规则。每跑一次 并行评估，最终取最高严重度。v4 update 把 Rule A 拆出 5 eV 上限 → blocked_by_bench，Rule B 接受 pbe_near_metal 软化：

规则触发条件处置
A · 已知金属 × 抽到 0.5–5 eV gap 开题判定为金属，但 bench 抽到中等带隙 fail + replan，提示加大 nbnd + 改 k-path 覆盖 Fermi 穿越点
A' · 已知金属 × 抽到 > 5 eV gap
v4 新增 · FIX-2		 物理不可能——只可能是 bench evaluator 在 alkali / 贵金属的 odd-Z 漏算金属 bug（Rb 案例） verdict='blocked_by_bench'，agent 直接 submit 不浪费 replan 预算；submission notes 里附 audit trail
B · 已知绝缘体 × 抽到金属 开题判定为绝缘体且 pbe_near_metal=False，但 bench 抽到金属 fail + replan，提示 NSCF 用 occupations='fixed' + 增 nbnd
B' · pbe_near_metal × 抽到金属

C · 已知 wide-gap × 抽到 < 0.05 eV 判定含 oxide/fluoride/wide-gap 但抽到几乎为 0 soft_warn（PBE 系统性低估，不强制重跑）
D · 低置信判定遇到物理矛盾 判定来源不是 deterministic 但发生 A 或 B 情形 soft_warn（判定本身不够硬，不强 replan）
E · 磁性矛盾 题目声明 main_magnetic_element 但 SCF 最终 total_magnetization < 0.1 µB fail + replan，提示 nspin=2 + starting_magnetization(X) 起点

📌 pbe_near_metal 列表（v4 新增 · FIX-3b）： config_advisor 在开题时给下面这 8 个材料打上 pbe_near_metal=True 标签， 本身不改 is_metal 判定（物理上仍是半导体），仅作为给 physics_sanity 看的 hint：

Ge（PBE gap ~0，实验 0.67 eV）· InSb（PBE gap ~0，实验 0.17 eV）· HgTe（PBE 半金属，实验半导体）· PbTe · PbSe · SnTe（IV-VI 窄带隙）· Bi₂Se₃ · Bi₂Te₃（拓扑绝缘体母体）。

规则 E / F 的阈值都是经验拍的：0.1 µB 判磁性塌缩、5 kbar (=0.5 GPa) 判残余应力。新加的 A' 用 5 eV 作为"物理不可能"上限。pbe_near_metal 列表当前 8 个材料—— 这些都对吗？还有哪些 metal/insulator 失配场景（Mott / Kondo / 半金属 / topological metal） 要不要写进规则？请教 → §6.3。

§4.4

把 QE 工作流切成几个独立阶段

我们把 DFT 工作流切成了 7 个阶段（"step kind"），planner 写计划时 只能从这 7 个里挑。每个阶段对应它自己的 system prompt 加注、自己的工具白名单、自己的验收清单。

阶段做什么
scf自洽场——给定结构 → 电荷密度 + scf.out
vc_relax变胞弛豫——晶格 + 内坐标全松，三阈值强制收紧
relax固定胞、只松内坐标。很少用，多数情况 vc_relax 就够
bandsk-path NSCF + bands.x 后处理，依赖上游 SCF 的电荷密度
dos密集 k-mesh NSCF + dos.x，依赖上游 SCF
collect纯文件操作（cp / 改名 / 合并数据）；禁止调 pw.x / bands.x / dos.x
physics_sanityT2 任务自动加在最后，做物理一致性 cross-check（见 §4.3）

切分的好处是每个阶段可以独立写自己的 prompt + 验收清单——SCF 那段 prompt 只讲电荷密度，bands 那段只讲 k-path 和 nbnd，互不污染；新加阶段就加一个模块。

但 bench 今年要扩到 phonon (DFPT) / 缺陷形成能 / 表面能 / NEB / HSE 杂化 / DFT+U 总能这些；当前 7 种是否够、要不要新增、粒度对不对（nscf 当前是嵌在 bands / dos 里的内部子步，要不要独立成一类？）——见 §6.4。

§4.5

在 pw.x 起跑之前自动改写输入文件的规则

每次 executor 准备执行 pw.x 调用，先把对应的输入文件 交给 lint 跑下面这几条规则；任何一条触发都会把输入改写到磁盘，并把 "改了什么 / 为什么"的 diff 块回写给 LLM——让它看见自己被改了什么，下次别再写错。

规则触发条件动作默认值 / 理由
A · vc-relax 三阈值 calculation='vc-relax' 取 min(LLM 写的值, 紧默认)——只能更紧不能更松 press_conv_thr=0.05（QE 默认 0.5）
forc_conv_thr=1.0d-4（QE 默认 1e-3）
etot_conv_thr=1.0d-5（QE 默认 1e-4）
理由：cubic 系统的 SpacegroupAnalyzer 对 ≥0.01° / 0.05% 漂移就降级
B · bands 必须 NSCF 模式 calculation='bands' + K_POINTS crystal_b/tpiba_b 改成 calculation='nscf' QE 只在 nscf 模式才写 highest occupied / lowest unoccupied 行， bench 靠这行抓 VBM/CBM
C · USPP ecutrho 比例 pseudo 含 USPP/GBRV/PAW 且 ecutrho/ecutwfc < 9 把 ecutrho 改成 10×ecutwfc GBRV USPP 标准 ~10×；4–8× 会让带隙漂 0.2–0.5 eV
D · occupations 与判定不一致 SCF / bands-NSCF 的 occupations 跟开题判定的金属/绝缘体不符 仅警告不自动改（开题判定在 Mott 绝缘体等边界 case 可能错） 金属 → smearing（mv, degauss=0.02）；绝缘体 → fixed

v4 wired into bash 主流程		 bands NSCF + K_POINTS crystal_b / tpiba_b 检查每段路径数据行：先是段数 N，随后 N 行 kx ky kz weight（4 列）；段数与行数不符 / 行少于 4 列 / weight 非整数 → 拒绝 spawn pw.x，把诊断回写给 LLM LLM 常把 seekpath 输出粘错 + 漏 weight 列；或者把高对称点标签也算进段数。 旧版本下 pw.x crash 给 "found ... lines, expected 8" 看不懂

处理策略有两条核心：

  1. 只收紧不放松——min(LLM 值, 紧默认)。假设是"如果题目能容忍松，那紧也不会更差， 多花 1-2 个 BFGS 步"。是否成立请教 §6.5。
  2. 不偷偷改，把 diff 回写给 LLM——下一轮 LLM 看到"我刚写的"和"最终落盘的"差， 自己学着把它写对。
§4.6

让 agent 自动累积"踩坑笔记"

前 4 条机制都是我们写死的 DFT 知识。 这一条不同——它让 agent 把"先失败、后修好"的经历当成一次教训攒下来， 下次遇到类似情况自动复用。但有个硬约束：教训只能是"方法学" （比如"vc-relax 早期力震荡时调小 mixing_beta"），不能是答案 （比如"Si 的 ecutwfc 用 80 Ry"）。

图 2 · 自动累积笔记的闭环。失败 → 修复 → 反思 LLM 提炼一条方法学教训 → 存进对应阶段的笔记本 → 下次同类问题直接塞进开场记忆。

为什么硬要"方法学 only"

我们最怕的不是 agent 学不会，而是它学会的是答案。 比如有人无意中存了一条"Si 体系 ecutwfc 用 80 Ry 才能让 gap 收敛到 0.012 Ha"， 下次再跑 Si 直接命中——看起来"学会了"，本质是答案缓存。bench 评的是 agent 的通用 DFT 能力 而不是查表能力。

所以在写进笔记本之前，每条候选教训过三层正则：

  1. 先把合法的"方法学陈述"屏蔽。比如 mixing_beta=0.3 这种 QE 参数值是方法学（你做 DFT 的人就要知道用什么参数），先 mask 掉再判断。
  2. 剩下文本里不许出现任何元素符号。118 个全集——"Si" / "Fe" / "O" 都拒。 禁止把任何物理"答案 trigger"绑到具体材料上。
  3. 不许出现 "数字 + 单位" 形状（eV / GPa / Å / Ry / bohr / kbar）。 一条教训如果说"gap = 1.2 eV"那就是答案缓存。

这三层一起，大概率拦住常见的 答案泄漏。但 LLM 可能用同义表达绕开——这是 §6.6 想请你看的。

每条教训贴什么标签（用于检索）

一条教训写进笔记本时带 4 个标签：

标签取值为什么这样切
任务类型 vc_relax (M1_T1) / bands_dos (M1_T2) T1 教训不该污染 T2 的 NSCF 设置
晶系 cubic / tetragonal / orthorhombic / hexagonal / trigonal / monoclinic / triclinic cubic 对 SpacegroupAnalyzer 漂移最敏感；六角对 c 轴 ecutwfc 异常更敏感； 分对称族避免"立方下的教训用到 triclinic"
原子数量级 ≤2 / 3–8 / 9–32 / >32 大单胞需要更细 mixing_beta；小单胞反之。按量级分桶避免桶太细 → 命中率低
工作流阶段 scf / vc_relax / bands / dos / collect / physics_sanity 每个阶段一本独立笔记，vc-relax 教训不会污染 bands 的 LLM 上下文

下次跑题时按这 4 个标签全匹配优先；如果 0 条匹配，回退到只匹配"任务类型 + 工作流阶段"。最多取最新 5 条注入。

什么时候触发反思

只在"先 fail 后 succeed"的阶段上触发——也就是 verifier 之前已经判失败、 后来某次重试通过了。一次性通过的"好运行"不写笔记（也许那是运气，提炼出来可能误导）。 触发后给反思 LLM 三样东西：

  • 失败那次 verifier 的"原因"（比如"BFGS 前 3 步力震荡"）
  • 失败和通过两次之间，executor 改了哪些文件、改了什么（diff，硬上限 4 KB）
  • 系统提示词里反复强调的硬约束（不准出现元素符号 / 不准带答案数值 / 一条教训 ≤ 200 字）

反思 LLM 如果觉得"这次没什么可教的"（比如失败那次只是打错了文件名）， 就主动 decline——空笔记本好过有毒的笔记本。

5 · 真实跑一题：Si vc-relax 大约 7 分钟里发生了什么

2026-05-18 跑的一道 Si 单元胞 vc-relax 任务，完整 trace 在 /srv/shared/science/web/runs_local/2026-05-18T11-56-54Z_M1_T1_Si_vc_relax/。 完整 408 s，bench 评分 1.0。下面用 4 段大白话讲讲整条决策链。

开场两道功课（约 15 s）

题目里说"Si 的 conventional cubic cell, 8 atoms"。两道前置功课：

  • 金属/绝缘体判定：Si 不在金属元素表里 → 判定为半导体，置信度"deterministic"。 建议 occupations='fixed'，ecutrho = 10×ecutwfc（USPP 标准）。
  • 结构预弛豫：M1_T1 任务不需要，跳过。

planner 写计划（约 8 s）

planner 读完题，输出一份只有 1 个阶段的计划：

steps = [
  { id: "vc_relax_1",
    kind: "vc_relax",
    deps: [],
    expected_outputs: { "vc_relax_out": "vc_relax_1/vc-relax.out" }
  }
]

就这么简单——M1_T1 一步就能搞定。

executor 干活 + lint 自动改写（约 6 分钟）

executor 开始写 QE 输入。它写了一版 vc-relax.in，里面：

  • press_conv_thr 没写（QE 默认 0.5 kbar）
  • forc_conv_thr = 1.0d-3（不够紧）
  • etot_conv_thr = 1.0d-5（已经够紧）

它准备 bash pw.x -in vc_relax_1/vc-relax.in > ... 那一刻，lint 拦下来：

──── 自动 lint 改写 ────
file: vc-relax.in
- press_conv_thr = (missing)   →   press_conv_thr = 0.05
- forc_conv_thr  = 1.0d-3       →   forc_conv_thr = 1.0d-4
原因：vc-relax 需要更紧的阈值（避免 cubic 系统的空间群退化）
────

diff 块回到 LLM 的对话里，pw.x 用改写后的输入 spawn。整段计算约 5 分钟， 最终 scf accuracy < 1e-9 + bfgs converged in 4 steps。

verifier 验收 + 提交（约 12 s）

verifier 用 vc_relax 阶段的清单去查：

  • ✓ JOB DONE. 在 log 末尾
  • ✓ bfgs converged in 4 steps
  • ✓ 最终 stress = 0.02 kbar < 0.05
  • ✓ cubic 系统：a = b = c = 5.4307 Å（差异 < 0.0001 Å）

verdict: pass。executor 交付 vc_relax_out = "vc_relax_1/vc-relax.out"， agent 写 result.json，bench 评分 1.0。

这条 trace 完整展示了 v4.2 的"纵深防御"——开场判定挡住 occupations， lint 在 spawn 之前补救阈值，verifier 事后查物理一致性。任意一层失守，其他两层还能兜住， 这就是 v4.2 比 v1 强的地方。

  • 普通验收现在大多是"文件存在 + 字符串匹配"层级。哪些真正的物理失败模式 我们没覆盖？例如：
    • SCF 形式上 JOB DONE. 但能量明显偏离（比经验值低 / 高几个 eV/atom）
    • vc-relax 最终 stress 分量都很小，但晶格 a/b/c 漂得严重
    • bands NSCF 的 VBM 跟 SCF 的 fermi 差超过 0.1 eV（说明 NSCF k-path 没覆盖真正的 VBM）
    • insulator 的 DOS 在 E_F 处不是 0
    • charge density 出现 NaN / spike
  • physics_sanity Rules E / F 阈值：total_mag < 0.1 µB 判磁性塌缩 fail； |P| > 5 kbar (=0.5 GPa) 判 vc-relax 残余应力 soft_warn。 这两个阈值选得对吗？
  • v4 新加的 A' 规则：advisor=metal × 抽到 > 5 eV gap → blocked_by_bench，agent 直接 submit 不浪费 replan（Rb 案例）。 5 eV 上限定得对吗？已知 bench 的 odd-Z 漏算金属 bug 在哪些其他 alkali / 贵金属上还可能发生（除了 Rb / Cs）？
  • v4 新加的 B' 规则 + pbe_near_metal 列表（Ge / InSb / HgTe / PbTe / PbSe / SnTe / Bi₂Se₃ / Bi₂Te₃ 8 个材料 soft_warn 而不 fail）。列表漏了哪些？ 候选 Ⅳ-VI / Ⅱ-Ⅵ / 重 SOC 体系挺多。还有哪些"DFT 系统性偏差"该加类似 flag？ 例如：underestimate magnetic moment / overestimate volume / SOC 系统性偏差 / PBE 高估磁性转变 Tc…
  • 其他金属/绝缘体失配场景（Mott / Kondo / 半金属 / topological metal）要不要也写进规则？ 哪些只该 soft_warn 哪些必须 fail？

§6.4 · 工作流切分粒度对不对

请审 §4.3 那 7 种阶段：

  • 现在的 7 种（scf / vc_relax / relax / bands / dos / collect / physics_sanity） 对应你日常 QE 工作流是不是自然切片？
  • nscf 现在嵌在 bands 和 dos 里，要不要独立成一类？ 好处是依赖关系更清晰；坏处是 plan 变长，planner 更难写对依赖。
  • bench 下半年要扩到 phonon (DFPT, ph.x) / 缺陷形成能 / 表面能 / NEB / HSE 杂化 / DFT+U 总能。每一类该怎么切？例如：
    • phonon 是不是 scf → ph.x（DFPT 计算 dyn matrix）→ q2r → matdyn 四步？
    • 缺陷形成能要不要拆 "perfect_scf + defect_scf + chempot" 三步？
  • "1 个阶段 = 1 个 LLM session" 这个粒度——你觉得太细还是太粗？

§6.5 · 自动改写输入的规则合不合理

请审 §4.4 那张默认值表（A 三阈值 / B bands→nscf / C ecutrho×10 / D occupations / E K_POINTS 格式）：

  • vc-relax 三阈值（0.05 / 1e-4 / 1e-5）——是否过紧？ 会不会有 cell 接近不稳定的体系（如近相变温度的钙钛矿）反而该用松阈值， 否则 BFGS 在 noise 里震荡不收敛？
  • "只收紧不放松"这个策略—— 有没有合法物理场景是"题目就需要松收敛"的？我们想过的反例只有 "做一次粗扫探索 EOS 形状"，bench 题目目前没这种。
  • USPP ecutrho ≈ 10×——是不是太一刀切？SSSP efficiency 里有些 PAW 其实 8× 就够， 有些 hard 元素（O / F）可能需要 12×；要不要按元素查表更细？
  • occupations 是"仅警告不强改"——你觉得开题高置信判定时应该强改吗？ 什么样的"合法反例"应该让 LLM 能 override？
  • Rule E（K_POINTS crystal_b 数据行校验）是新加的，逻辑没人审过： 拒绝 spawn 然后回写诊断 vs 自动改写 vs 仅警告，哪个动作更合理？ K_POINTS automatic / K_POINTS gamma 是否也该加 sanity？
  • 有没有其他规则该加但我们没加的？例如：
    • 磁性体系自动注入 nspin=2, starting_magnetization 起点
    • 含 d/f 元素自动给个 U_value 候选值
    • k-mesh 太稀（小于 8×8×8 in 立方）自动加密
    • 大单胞自动调 mixing_beta / diagonalization='davidson'

§6.6 · 让 agent 累积经验的边界

请审 §4.5 整节——这是 05-19 加的新机制，所有边界都是我一个人定的，最需要 DFT 同行的眼睛。

  • "方法学 only" 的边界：
    • 3 层正则（元素符号 / 答案字段 / 数字+单位）能拦住"在 Si 上把 ecutwfc 调到 80 才让 gap 收敛"这种伪装成方法学的答案缓存吗？我们怕的是 LLM 学会用同义表达绕开 （比如"原子序数 14 的体系"代替 Si）。
    • QE 参数白名单只覆盖 20 来个常用参数；Hubbard U / 磁性 / DFT+D 系列、ph.x 输入参数 都没加。要不要扩成两表（核 + 扩展，扩展只在对应任务类型触发）？
    • "数字+单位一律视为答案"对不对？合法反例： "超过 0.5 GPa 的残余应力"这种"阈值描述"算方法学还是答案？ 当前规则会被误判成答案。
  • 4 个标签（任务类型 / 晶系 / 原子数量级 / 工作流阶段）够不够？ 物理上是否漏了：
    • spin / magnetism（有磁 vs 无磁，教训共享性差异巨大）
    • 元素组成（含 d/f / 含轻 / 含重 / 纯 sp）
    • 赝势家族（GBRV USPP vs SSSP-mixed vs PseudoDojo NC）
    • 电子相关强度（Hubbard U / hybrid / DMFT...）
    维度多 → 命中率低，维度少 → 教训误用。你的 DFT 经验里，哪几个维度上同一条教训 "不能跨"？
  • 教训长寿命问题：笔记本永远 append、不过期。一条 2026 年的教训用 到将来 QE 7.6（假设默认 smearing 算法变了）还成立吗？要不要：
    • 带 QE 版本戳，按版本过滤？
    • 每条教训带"命中计数"，命中 N 次没失效再升权重？
    • 接受 stale 风险（反正反思还能写新教训覆盖旧的）？
  • 触发条件偏严：只在"先 fail 后 pass"的阶段触发反思。一次性 pass 的"好运行" 不写笔记。你觉得：
    • 好运行也该提炼"关键配置回放"（哪些参数选对了）？
    • 还是这种过度提炼反而容易蜕变成答案缓存？
