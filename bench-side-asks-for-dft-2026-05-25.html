议题 1 · 金属短路阈值 1e-3 BENCH 请决策

现象

T2 bands_dos 评估器决定"要不要把材料短路成金属"的判据是一个写死的 1 meV 阈值：

# /srv/shared/science/bench/harness/lib/evaluator.py:697 与 :750
        gap_value = max(0.0, cbm_e - vbm_e)
        if gap_value < 1e-3:                       # ← 1 meV
            return _metal_t2_result(dos_e_fermi, dos_curve)

证据：两个材料骑在这条边上

材料GT 标agent 实算 gap评估器判定后果
Ge（金刚石）metal（PBE 实算近零隙，诚实反映） 0.0088 eV（k-mesh 依赖，0~10 meV 抖） 0.0088 > 1e-3 → direct 3 个标签字段全失分 → 0.6（曾经 agent 报 0.0000 触发短路时拿 0.956）
Rb（半芯金属）metal 0.020 / 0.043 eV（2026-05-22 三次新跑实测） > 1e-3 → direct/indirect gap_type / 标签失分 → 稳定 0.64
注意 Rb 是复核时的新发现：原报告把 Rb 归到 FIX-1（奇数电子伪能隙，10.989 eV）。 那个 bug 在权威仓里已经没了（见 §5）；Rb 现在的残留失分恰恰 落在同一个阈值问题上——只不过它的实算 gap 是 20~43 meV，比原报告 FIX-3a 提的 10 meV 还高。所以单纯把阈值从 1 meV 抬到 10 meV，能救 Ge，救不了 Rb。
需要你们决策： "PBE 实算近零隙"和"半芯金属残留小隙"这两类材料，正确的金属判据是什么？
  • 简单方案：阈值 1e-3 → 1e-2（10 meV），吸收 PBE 数值噪声（Ge / α-Sn / InSb / HgTe / PbTe 一族）。但 Rb 的 20~43 meV 仍漏。
  • 更稳方案：当 GT 的 is_metal 标为真时，用 DOS(EF) > 0 或 GT 自身 metal 标记直接判金属，不依赖 agent 提取的 gap 数值。
  • 风险：阈值抬太高会把真实的窄带半导体误判成金属——这正是需要你们物理判断的地方。

原报告依据：§3 Case B — Ge（FIX-3a）。本页 Rb 残留数据为 0525 复核新增。

议题 2 · DOS per-fu vs per-cell 归一化 BENCH 请复核+合并

现象

GT 的 DOS 参考文件 header 明确标的是 per formula unit：

# E_eV(relative_to_VBM)  DOS_states_per_eV_per_fu
-4.993400  1.625000

但 QE dos.x 输出按惯例是 per primitive cell。BNhex 的 hex 原胞 = 4 原子 = 2 f.u.， 所以 agent 数值天然是 GT 的 ~2 倍。评分链路只做能量轴平移、不除 n_fu：

# /srv/shared/science/bench/harness/lib/evaluator.py:683（_build_dos_curve 返回）
        return {"energy_ev": energies_rel, "dos": list(dos["dos"])}
                                          # ← 没除 n_fu

影响

评分用 F1 面积重叠、曲线是面积归一化的，所以 2× 标签错只被部分吸收（不会直接归零， BN / NaCl 这种还能拿 ~0.63），因此一直藏着没暴露。但 web 端 result 页画的是原始数值， 视觉上 agent 曲线"明显比 GT 高一截"——一眼能看出协议没对齐。

建议改法 在 _build_dos_curve 把 dos 除以 atoms_per_primitive_cell / atoms_per_fu。 数据是现成的——harness 已经在用 atoms_per_primitive_cell（见 evaluator.py:935，T3 EOS 的 per-atom rms）， 只差把 atoms_per_fu 接进 T2 dos 评分路径。
需要你们确认： atoms_per_fu（或等价的 formula-unit 数）在评分时从哪取——题目 JSON？结构解析？ 确认来源后这就是一行 diff，我可以直接提 PR 给你们复核。

原报告依据：§9.3 Case E — BNhex Layer 2。

权威仓 questions/M1_T2/M1_T2_BN_hex_bands_dos.json 同时含两条相互冲突的信息：

"convergence_evidence": {
  "dos_params": { "sigma_ev": 0.02, "de_ev": 0.01 },   ← 暗示 Gaussian σ=0.02

},

  "gt_quality":     "self_run_converged",
  "gt_switched_at": "2026-04-30"                        ← 切到 self-run tetrahedra

实际 GT 生成脚本（scripts/regen_t2_dos_with_tetrahedra.py）不指定 ngauss / degauss， 让 dos.x 复用 NSCF 的 tetrahedra 积分。换句话说，dos_params.sigma_ev=0.02 是 2026-04-30 切换前残留的反事实字段。

已无碍我侧 我侧 solver 的 prompt 已经更新对齐了（/srv/shared/science/solver/v4.2/agent/system_prompt.py:643-652 现在写 "GT switched from MP-Gaussian to self-run-tetrahedra" + "DO NOT set ngauss or degauss"）。 所以这条不再卡我们——但题目 JSON 里那个误导字段还在，对任何读 GT 元数据的人（或下一个 solver）都是坑。
需要你们决策+改： 在题目 JSON 里显式声明 GT 的 DOS 积分协议，例如加 "dos_method": "tetrahedra" 字段；并把误导的 dos_params.sigma_ev 删掉或标注为 "historical (pre-2026-04-30)"。让 agent / 复核者一眼能看出 GT 真实协议。

原报告依据：§9.2 Layer 1 + §9.6 讨论点 1。

议题 4 · dos_curve 评分指标是否仍合适 BENCH 请决策

当前 dos_curve 子项用 F1 面积重叠。问题：它对"协议偷懒"的容忍度太高—— 一个积分协议选错、归一化也错的 DOS（BN / NaCl 这类），形状大致对就还能拿 0.63。 等价于"形状对就行、协议错不扣分"。

需要你们决策（方法学）： 要不要换更严格的指标？候选：
  • KL divergence（对峰位 / 峰宽更敏感）；
  • 带 bandwidth penalty 的 shape similarity；
  • normalized cross-correlation（先归一化再比形状，能和议题 2 的归一化解耦）。
这条和议题 2（归一化）相关但独立：归一化是"对齐绝对高度"，这条是"形状判据本身够不够严"。

原报告依据：§9.6 讨论点 3。

5. 已解决（无需动作）· Rb 奇数电子伪能隙 RESOLVED

原报告 §2 的 FIX-1：Rb（GBRV USPP，Zval=9，半芯 4s²4p⁶5s¹）在 'mv' smearing 下 QE 不打印 homo/lumo，评估器走 _find_vbm_cbm_via_electron_count 按电子数切带，round(9/2)=4 → 把 4p 半芯顶当 VBM、5s 金属带底当 CBM → 报出 10.989 eV 伪能隙 → 3/3 失败。

复核结论：权威仓里这个 bug 已经不存在

权威仓 /srv/shared/science/bench 的 VBM/CBM 定位只剩两条路径—— _find_vbm_cbm_via_homo_lumo（:485）和 _find_vbm_cbm_via_fermi（:521）。 原报告 FIX-1 要打的 _find_vbm_cbm_via_electron_count 函数整个被重构掉了， Rb 这种情况现在直接走 fermi-slice。

实跑数据印证（state.db，权威 bench 打分）：

Rb bands_dos:
  2026-05-19 / 05-21    failed         ← 原报告的伪能隙时期
  2026-05-22 07:08      done   0.642   ← 已恢复
  2026-05-22 12:54      done   0.641
  2026-05-22 14:35      done   0.642

提取的 band_gap_ev：  0.020 / 0.043 eV   （← 不再是 10.989 eV）

Rb 在 2026-05-22 从 failed 翻成 done ~0.64，提取 gap 是 0.02~0.04 eV（不是 10.989）—— 说明 fermi-slice 路径已正确处理半芯金属。FIX-1 因此作废，不进本次讨论。 它残留的那点失分（小隙被标 direct/indirect）归到议题 1。
