议题 1 · 金属短路阈值 1e-3 BENCH 请决策

现象

T2 bands_dos 评估器决定"要不要把材料短路成金属"的判据是一个写死的 1 meV 阈值：

# /srv/shared/science/bench/harness/lib/evaluator.py:697 与 :750
        gap_value = max(0.0, cbm_e - vbm_e)
        if gap_value < 1e-3:                       # ← 1 meV
            return _metal_t2_result(dos_e_fermi, dos_curve)

核心矛盾（本议题要害）：连续量与分类量对"gap 是不是零"用了相差 100× 的标准

同一个 T2 评测，对"这个 gap 算不算零"这个同一物理问题，给了两套互相打架的判据：

  • 连续量 band_gap_ev（absolute rule，容差 ±0.1 eV = 100 meV）：算出的 gap 落在 0.1 eV 内就算命中 GT 的 0.0 —— 等价于评测器在说"≤ 100 meV 都当零、基本算金属，接受"。
  • 分类量 gap_type / vbm_k_label / cbm_k_label（exact rule）：要被标成 metal，gap 必须 < 1 meV（evaluator.py:697 主路径 / :750 fallback 的短路阈值），否则就走 seekpath 给出真实的 direct/indirect + Γ 等 k 标签。

两条判据对"零"的界定差了 100×（100 meV vs 1 meV）。于是当 PBE 把窄隙材料算成几~几十 meV 的小而非零的 gap 时，同一个数会同时满足连续量（"≈ 0，匹配 metal GT"）又踩中分类量（"> 1 meV，是 direct 半导体"）—— 评测器自相矛盾，3 个分类字段必然连锁失分。这就是分类标准与连续量标准不一致导致的失分，不是 agent / QE 算错。

Ge 实锤（M1_T2_Ge_bands_dos · run 2026-05-22T14-53-56Z · report.json · total_score = 0.635）—— 注意所有行都源于同一个 0.023 eV：

字段GTrule / 容差agent 实测对 0.023 eV 的裁决得分
band_gap_ev0.0absolute / ±0.1 0.023 eV|0.023−0| = 0.023 < 0.1 → 当作 ≈ 01.0 PASS
gap_typemetalexact（短路要 < 1 meV） direct23 meV > 1 meV → 判半导体0.0 FAIL
vbm_k_labelmetalexact GAMMA≠ metal0.0 FAIL
cbm_k_labelmetalexact GAMMA≠ metal0.0 FAIL
dos_e_fermi7.7839absolute / ±0.1 7.7983差 0.014 < 0.11.0 PASS
dos_curve—F1 形状 —形状基本对0.93

同一个 0.023 eV：连续量判据给满分（把它当 ≈ 0 / 金属），分类量判据判 0 分（把它当直接带隙半导体）。3 个分类字段集体归零，把 total 从满分拖到 0.635。该 run 的 notes 字段已自我诊断到这一点："…smearing 路径下 fermi-splitter eps=5 meV 容差未能覆盖…仍跨过 evaluator 1 meV 阈值…等 bench 侧 FIX-3a（主路径阈值 1e-3 → 1e-2）"。

区分两个量 fermi-splitter 里的简并容差 eps = 5e-3（5 meV，evaluator.py:524）跟这里的短路阈值 1e-3 是两个不同的量：eps 是"把哪些带归到 VBM/CBM 候选"的分带容差（吸收 bands.dat 1 meV 舍入噪声）；1e-3 是"算出来的 gap 小到什么程度就当金属"的判零阈值。本议题谈的是后者；前者在 Ge 这例里没能把 23 meV 的近简并带兜住，所以也没救成。

同根的第二例：Rb（说明单抬阈值不够）

材料GT 标agent 实算 gap评估器判定后果
Rb（半芯金属，非 pbe_near_metal）metal 0.020 / 0.043 eV（2026-05-22 三次新跑实测） > 1e-3 → direct/indirect gap_type / 标签失分 → 稳定 0.64
注意 Rb 是复核时的新发现：原报告把 Rb 归到 FIX-1（奇数电子伪能隙，10.989 eV）。 那个 bug 其实没修、只是不影响官方分（见 §6）；Rb 现在的残留失分恰恰 落在同一个阈值问题上——只不过它的实算 gap 是 20~43 meV，比原报告 FIX-3a 提的 10 meV 还高。所以单纯把阈值从 1 meV 抬到 10 meV，能救 Ge，救不了 Rb。
需要你们决策： "PBE 实算近零隙"和"半芯金属残留小隙"这两类材料，正确的金属判据是什么？
  • 简单方案：阈值 1e-3 → 1e-2（10 meV），吸收 PBE 数值噪声（Ge / α-Sn / InSb / HgTe / PbTe 一族）。但 Rb 的 20~43 meV 仍漏。
  • 更稳方案：当 GT 的 is_metal 标为真时，用 DOS(EF) > 0 或 GT 自身 metal 标记直接判金属，不依赖 agent 提取的 gap 数值。
  • 风险：阈值抬太高会把真实的窄带半导体误判成金属——这正是需要你们物理判断的地方。

原报告依据：§3 Case B — Ge（FIX-3a）。本页 Rb 残留数据为 0525 复核新增。

现象

-4.993400  1.625000

# /srv/shared/science/bench/harness/lib/evaluator.py:683（_build_dos_curve 返回）

影响

需要你们确认： atoms_per_fu（或等价的 formula-unit 数）在评分时从哪取——题目 JSON？结构解析？ 确认来源后这就是一行 diff，我可以直接提 PR 给你们复核。

议题 4 · dos_curve 评分指标是否仍合适 BENCH 请决策

当前 dos_curve 子项用 F1 面积重叠。问题：它对"协议偷懒"的容忍度太高—— 一个积分协议选错、归一化也错的 DOS（BN / NaCl 这类），形状大致对就还能拿 0.63。 等价于"形状对就行、协议错不扣分"。

需要你们决策（方法学）： 要不要换更严格的指标？候选：
  • KL divergence（对峰位 / 峰宽更敏感）；
  • 带 bandwidth penalty 的 shape similarity；
  • normalized cross-correlation（先归一化再比形状，能和议题 2 的归一化解耦）。
这条和议题 2（归一化）相关但独立：归一化是"对齐绝对高度"，这条是"形状判据本身够不够严"。

原报告依据：§9.6 讨论点 3。

议题 5 · dos_e_fermi 绝对容差 0.1 eV 对 layered vdW 偏紧 BENCH 请决策

现状：评分规则

dos_e_fermi 走 absolute rule，容差 0.1 eV（题目 JSON 的 ground_truth.dos_e_fermi）。 打分实现 /srv/shared/science/bench/harness/lib/scoring.py:187-216（_score_with_tolerance）：

err = |agent − GT|
err ≤ 0.1            → 1.0
0.1 < err ≤ 0.2      → 线性衰减 1.0 → 0.0   （partial credit 开时）
err > 0.2            → 0.0

注意对绝缘体 / 半导体，按 杨博 2026-05-18 的约定 dos_e_fermi 取的是 VBM（带顶）而非真费米能（evaluator.py 在 is_metal_per_gt=False 时设 dos_e_fermi=vbm_e）。 所以这个字段实际在比"绝对 VBM 能量"。

典型材料：BNhex（唯一进 T2 的 layered vdW）

样本：M1_T2_BN_hex_bands_dos，runs 2026-05-22T00-13-26Z / 14-44-15Z。

量GTagent（两次跑）err得分
dos_e_fermi（= VBM）2.4533 eV 2.8123 / 2.8131 eV 0.359 eV 0.0（> 2×tol，部分分也没）

两次跑的 agent 值（2.8123 / 2.8131）彼此只差 0.8 meV——高度可复现，说明这不是 agent 抖动， 而是 agent 与 GT 之间一个稳定的 0.36 eV 系统性偏移。BNhex 综合分 0.80 里，dos_e_fermi 这 0 分是失分项之一 （另一项 dos_curve 见议题 2 / 3）。

把机理说准 agent 和 GT 都是 PBE，所以这不是"PBE 对实验的 0.1–0.2 eV 误差"那回事——那个误差在两边会抵消。 0.36 eV 这种量级、且跨 run 高度可复现的偏移，更像是绝对能量零点 / VBM 参考系不一致 （周期 DFT 里 VBM 的绝对值挂在内部平均势上，layered vdW 带真空层时尤其敏感），叠加面外 k-mesh 对 EF/VBM 收敛的松动。 关键推论：单把容差从 0.1 放宽到 0.2 救不了 BN（0.36 > 0.2）——跟议题 1 的阈值是一类问题。

需要你们决策： 对 layered vdW（及一般绝缘体）这类材料，dos_e_fermi 该怎么比？
  • 简单方案：把绝对容差从 0.1 放宽（如 0.2~0.3 eV）。但 BN 的 0.36 eV 仍漏，且对真费米能（金属）放太松会失去区分度。
  • 更稳方案：改成 referenced 比较——既然 DOS 曲线本来就以 VBM 对齐（relative_to_VBM），dos_e_fermi 字段对绝缘体也应比"相对量"或干脆不单独记绝对 VBM；金属才比真 EF。这样消掉绝对零点漂移。
  • 需要你们物理判断：layered vdW 的 EF/VBM 在你们的 GT pipeline 里参考的是什么零点？agent 要对齐到什么才公平？

数据：BNhex run report.json（per_field_scores.dos_e_fermi=0.0）+ scoring.py:187 / evaluator.py VBM 参考切换（杨博 2026-05-18）。本议题为 0525 新增。

6. Rb 奇数电子伪能隙 · run 已不再失败（solver 绕过），但 bench bug 未修 不阻塞 · 无需 DFT 动作

原报告 §2 的 FIX-1：Rb（GBRV USPP，Zval=9，半芯 4s²4p⁶5s¹）在 'mv' smearing 下 QE 不打印 homo/lumo，评估器走 _find_vbm_cbm_via_electron_count 按电子数切带，round(9/2)=4 → 把 4p 半芯顶当 VBM、5s 金属带底当 CBM → 报出 ≈ 11 eV 伪能隙 → 原先 3/3 失败。

关键：有两个 evaluator 同时在跑，行为不同

odd-nelec bug 并没有被修，它仍活在 ~/projects/bench/harness/lib/evaluator.py:517 （n_val = int(round(nelec/2))，由 commit 898cdf4 "electron-count fallback" 加入）。 但官方打分用的是另一份 bench，两条路径各取一个 evaluator：

solver physics_sanity 自检  →  import ~/projects/bench 的 evaluator  →  electron-count  →  10.988 eV（bug 仍触发）
官方打分 (web backend)      →  config.toml bench_dir=/srv/shared/science/bench  →  fermi-slice  →  0.020 eV

权威仓 /srv/shared/science/bench 从来没有 _find_vbm_cbm_via_electron_count （git log -S 该函数名在该仓历史中为空），它的 VBM/CBM 定位只有 _find_vbm_cbm_via_homo_lumo（:485）+ _find_vbm_cbm_via_fermi（:521）。 所以官方分数从来不受这个 bug 影响。

让 run 不再失败的是 solver 侧 FIX-2（commit e7c66d2，2026-05-20 09:40 UTC， 不是任何 bench 改动）。它给 physics_sanity 加了一条：当 "advisor=metal × 提取 gap > 5 eV" （真实金属物理上不可能）时，识别为 odd-nelec bug 的签名，判 blocked_by_bench 并放行提交，不再 replan：
