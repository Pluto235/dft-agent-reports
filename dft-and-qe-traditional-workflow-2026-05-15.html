§2 DFT 是啥、能算啥

一句话：DFT 是一种从量子力学第一性原理出发，预测"原子摆好后电子云怎么排"的算法。 输入是原子的位置，输出是电子的分布、体系总能量、原子受力、电学和光学性质（能带、态密度、磁性...）。

你可以把 DFT 想成一个超精确的物理模拟器。给它一组原子位置 → 它告诉你 "假如这些原子按薛定谔方程演化、达到稳态后会长啥样"。 你不需要懂量子力学就能用它——就像你不需要懂 Linux 内核就能用 cat。

DFT 能算出来的物理量很多。你 bench 当前涉及的主要是这几类：

物理量研究员关心啥对应你 bench
总能量（total energy）比较哪个结构更稳定vc-relax 用到
原子受力（forces）判断结构是否优化好（力小=稳定）T1 vc-relax 的收敛信号
应力（stress）判断晶胞是否要变形T1 vc-relax 收敛信号
能带（band structure）判半导体/金属、看 band gapT2 第一段
态密度（DOS）电子在不同能量上的分布T2 第二段

§3 Quantum ESPRESSO 是谁

QE 是开源的 DFT 软件包，意大利团队主维护，Fortran 写的，跑在 HPC 集群上。 你 bench 用 QE 是因为：开源、HPC 友好、社区教程多、有 GPU 加速版。

DFT 软件就像编程语言——VASP（商业、贵）、Quantum ESPRESSO（开源、最广）、ABINIT、GPAW... 本项目选 QE，理由跟"为啥选 Python 不选 MATLAB"差不多：免费、社区好、能跑大算例。

你需要认识的 QE 二进制（agent 通过 bash 工具调用）：

二进制干啥对应 bench
pw.x核心——跑 SCF / NSCF / 结构优化T1+T2 都用
bands.x把能带计算的结果整理成画图友好的 .dat 格式T2
dos.x把 NSCF 结果积分成态密度T2

§5 6 张概念卡片

不用一次全记。下面 §6-§9 用到哪个回来查就行。

💡 1/6

Plane-wave basis · ecutwfc · ecutrho

ecutwfc 就像照相机像素—— 太低 = 照片糊（算出来的电子云模糊、能量不准）； 太高 = 文件大 + 处理慢（CPU 时间爆炸）； "合适"值有个门槛，过了门槛再加像素只是浪费。

技术层面：QE 把电子的波函数展开成平面波叠加。ecutwfc（波函数截断能，Ry）决定用多少个平面波， 越大越精确越慢。ecutrho 是电荷密度截断，USPP 一般取 ecutwfc 的 8 倍。

Si 用 ecutwfc=60、ecutrho=480。这些值不是随便挑的—— 有人事先做了"收敛测试"确认对 Si 类材料够用。§7 末尾有详细的"太低会怎样"对照表。

💡 2/6

Pseudopotential（赝势）

原子的"内层电子"离原子核近，行为像快递盒里"重又没用"的填充物——参与不了化学键合， 但用平面波展开它要花很多算力。赝势就是把"原子核 + 内层电子"打包替换成一个等效势， 只对"外层（价）电子"算。等于把不参与的部分黑盒掉。

技术层面：赝势是个文件（.UPF 格式）。三种主流类型： NC（norm-conserving，最严）/ USPP（ultrasoft，最常用）/ PAW（VASP 主流）。 本项目用 GBRV USPP（PBE 配套）。赝势在 bench 里是写死的——只有 GBRV 一套，agent 没法换。

Si 的赝势文件叫 si_pbe_v1.uspp.F.UPF。agent 用 ls pseudos/ 找到这文件名， 在 QE 输入里引用即可。

💡 3/6

k-mesh · Brillouin Zone

晶体的电子状态分布在一个叫 "Brillouin Zone"（倒空间的"原胞"）的三维区域里。 要算性质就得在这个区域里采样——k-mesh 就是 BZ 上的采样网格，类比给一张 3D CT 扫描设几层。 8×8×8 = 512 个采样点；越密越准也越慢。

实用法则：金属需要密 k-mesh（16×16×16+，因为 Fermi 面要细化）； 绝缘体粗一点（6×6×6）也够；大单胞（4+ 原子）可以稀疏。

Si bulk 用 8×8×8 就够。后面 §9 算 DOS 时会加密到 24×24×24，因为 DOS 需要细致采样。

💡 4/6

SCF 自洽循环

SCF（Self-Consistent Field）解决一个"鸡生蛋蛋生鸡"问题：电子密度依赖势场、势场又依赖电子密度。 QE 的做法：猜个初始密度 → 算势场 → 解方程 → 得新密度 → 跟旧的混合 → 重复， 直到两次迭代结果几乎一样（"收敛"）。

典型现象：Si bulk 4 次迭代就收敛了（绝缘体好脾气）；金属 / 磁性体系可能要 30-50 次甚至发散； 不收敛是 DFT 第一大常见错误。

收敛信号：QE 输出里写 "convergence has been achieved in N iterations" 就是成功。 没收敛会写 "convergence NOT achieved"。

💡 5/6

vc-relax · 结构优化

"vc-relax" = variable-cell relaxation。同时让原子移动 + 让晶胞变形，直到所有原子受力为零 + 晶胞应力为零。 类比：你拿一团揉皱的橡皮泥（粗略原子排布），松手让它自己舒展回最稳定的形状。

实用层面：T1 题目就是 vc-relax。关键三个收敛阈值（etot/forc/press_conv_thr）必须够严， 否则结构没完全舒展、晶格偏离对称、空间群被 pymatgen 降级。v3 的 lint hook 自动收紧这三个阈值就是为这个。

Si vc-relax 跑完得到原胞向量长度 a = 3.866 Å，空间群编号 227 (Fd-3m，立方)。

💡 6/6

Band gap · DOS · Fermi energy

晶体里电子按能量分层占据。VBM（最高占据态）跟 CBM（最低空态）之间的能量差叫 band gap。 有 gap 是半导体/绝缘体，没 gap 是金属。Fermi energy 是"零温下电子能爬到的最高能量"。

态密度（DOS）是另一种看法：横轴能量、纵轴该能量上"能放多少电子态"——一条曲线。

direct vs indirect gap：如果 VBM 和 CBM 在同一个 k 点 → direct gap（半导体能直接发光）； 不在同一个 k 点 → indirect gap（如 Si，做不了 LED）。

Si 是经典 indirect gap：VBM 在 Γ 点，CBM 在 BZ 内部 OFF_PATH 位置；band gap = 0.62 eV（PBE 算的，实验 1.12 eV）。 T2 评分要 agent 报对这几个值。

§7 Si bulk · 结构优化（对应 T1）

SCF 是"固定原子算电子"。vc-relax 是在 SCF 外面套一个优化循环—— 每次 SCF 跑完看力 + 应力，调整原子位置 + 晶胞，再跑下一轮，直到力 / 应力都低于阈值。

vc-relax 输入跟 SCF 比，主要新增三件事：

  1. calculation = 'vc-relax'
  2. 新增 &IONS、&CELL namelist（指定优化算法 BFGS）
  3. 三个关键收敛阈值：etot_conv_thr = 1e-5、forc_conv_thr = 1e-4、press_conv_thr = 0.05 kbar

7.1 跑完看什么

7.2 ⚠ 易误解：ecutwfc / k-mesh 不是调速旋钮

很多人第一次会以为"参数越大越准、越小越快——选 60 是为了速度跟准确度的平衡"。这是错的。 正确说法：这些参数有一个"收敛门槛"——低于门槛结果就错了，高于门槛只是浪费时间。

给你看 Si 的真实数字（不同 ecutwfc 算出来的晶格常数）：

ecutwfc (Ry)Si 晶格常数 a 算出值结论
20~5.55 Å❌ 错（偏差 > 2%，T1 直接 fail）
30~5.46 Å❌ 还不够准
40~5.43 Å⚠ 接近收敛
605.43 Å✅ 过门槛，结果稳定
805.43 Å⚠ 不变了，多花一倍时间
1205.43 Å⚠ 大浪费

所以 "ecutwfc=60" 不是随便填的——有人提前做了收敛测试验证 Si 类材料够用。这些"够用值"被写进 v3 prompt 里 让 agent 闭眼抄。

agent 在 T1 这步容易翻车的几点：
  1. 三阈值忘写或写松 → 结构没充分舒展 → 空间群被 pymatgen 降级到三方 166 → 扣 0.25 分。
    ✅ v3 修了：qe_input_lint 钩子拦截 bash 命令，在 pw.x 启动前自动改输入文件。
  2. 跑完忘了用 pymatgen 验证：只看 JOB DONE 就 submit，没意识到空间群不对。
    ✅ v5 修了：MCP 工具 parse_structure + verify_run 强制跑。
  3. nstep=100 不够用：复杂体系（CsPbI3、HfO2）几十步还在抖。
    ⏳ 未修：v5 加 budget-aware termination 后才能优雅地"交个不完美的"。

§8 Si bulk · 能带（T2 第一段）

能带计算必须分两步跑——这是 QE 跟其他软件区别最大的地方之一。

两步逻辑就像盖房子 + 装修：第一步（SCF）打地基——用 8×8×8 均匀网格算出体系的电荷密度， 存到磁盘。第二步（NSCF on path）走样板间——不再自洽迭代，固定上一步的电荷密度， 沿着 BZ 一条特定路径（如 Γ → X → L → Γ）取 ~240 个 k 点，每个点求本征值。 没第一步的地基，第二步无从开始。

8.1 三步走

  1. SCF on 8×8×8 网格（同 §6）→ 产出 out_scf/si.save/charge-density.dat
  2. NSCF along path：用 seekpath 自动给一条 fcc 标准路径（241 个 k 点），跑 pw.x 并设 calculation='bands' + nbnd=16（多算些空带，能看到导带）
  3. bands.x 后处理：把 NSCF 结果整理成画图友好的 bands.dat（每行是 k_distance + 16 个能带值）

8.2 拿 bands.dat 找 VBM / CBM

  1. 每个 k 点 16 个能带按能量升序排，前 4 个是价带（占据），后 12 个是导带（空）
  2. VBM = 所有 k 点中"第 4 个能带"的最大值，记下它在哪个 k 点（Si 是 Γ）
  3. CBM = 所有 k 点中"第 5 个能带"的最小值，记下它在哪个 k 点
  4. Si 的 CBM 在 BZ 内部 (0.85, 0, 0.85) 附近——不在 high-symmetry path 上，标记为 OFF_PATH
  5. Band gap = CBM − VBM = 0.62 eV（PBE 算的；实验 1.12 eV 因为 PBE 系统性低估）
  6. Gap type = indirect（VBM 和 CBM 不同 k 点）

  2. nbnd 选少了：默认只算占据态 = 4 → 导带 0 条 → 找不到 CBM。
    v3 prompt 写了 "nbnd = 2× 占据 + buffer"。
  3. VBM/CBM 的 k 标签报错：Si CBM 在 OFF_PATH 不在标准点上——agent 容易把它强行说成"最近的标准 k 点"。 没现成自动化，需要后处理判 "距离 < tolerance 才报 label"。

§9 Si bulk · 态密度（T2 第二段）

态密度（DOS）跟能带需求的 k-mesh 完全不一样——bands 是沿路径稀疏，DOS 要整个 BZ 均匀采样越密越准。 所以再跑一次 NSCF，但参数不同：

  • calculation = 'nscf'（注意这次写 'nscf' 不是 'bands'）
  • K_POINTS 用 automatic 24 24 24（比 SCF 的 8×8×8 密 3 倍）
  • 跑完后用 dos.x 后处理，产出两列文件 dos.dat：能量 + DOS 值
▶ 展开看：dos.in 后处理 
&DOS
  prefix = 'si'
  outdir = './out_scf'
  fildos = 'dos.dat'
  Emin = 1.19                  ! Fermi - 5 eV
  Emax = 11.19                 ! Fermi + 5 eV
  DeltaE = 0.01
/

从 scf.out 里 grep "Fermi energy" 一行就能拿到 e_fermi 数值。Si bulk 是 ~6.25 eV（PBE）。

T2 第二段评分字段Si 期望值
dos_e_fermi6.254 eV（±0.1）
dos_curveFermi ±5 eV 范围的曲线，跟参考 L2 距离 ≤ 0.1
DOS 这步 agent 最常见的坑：
  • DOS k-mesh 不够密：8×8×8 直接跑 dos.x → 曲线锯齿 → L2 距离过大扣分。需要 16-24 倍。
  • 忘了跑 dos.x：只跑 NSCF 就 submit，dos.dat 不存在 → 评分系统读不到 → 0 分。 v5 的 verify_run gate 强制查文件存在。

§11 常见失败模式

研究员见多了的"坑"——agent 也会踩。下面是 bench 实测里出现过的 6 类：

症状原因 + 经验解决方法对应 agent 设计
SCF 不收敛 金属 / 磁性体系 mixing_beta 太大（默认 0.7）→ 降到 0.3-0.4；加大 smearing；改 mixing_mode='local-TF' v5 子 agent convergence_fixer 跑 6 条 playbook
vc-relax 完了空间群从 227 降到 166 力 / 应力阈值太松 → 用更严的 1e-4 / 0.05 kbar v3 qe_input_lint 钩子自动收紧

磁性体系算出无磁矩 没设 starting_magnetization → 显式给初始磁矩打破对称 v3 prompt 提；agent 自己判磁性还没做

§13 什么是写死的 / 什么是 agent 自由的

这一节直接回答"agent 在你 bench 里有多大自由度"——三色分级。

🔴 红色：写死的（agent 完全没法改）

项由谁定 / 在哪
题目集（148 题）~/bench/dft-agent-bench/questions/*.json
每题任务类型（T1 vc_relax / T2 bands_dos）题目 JSON
初始结构（晶胞 + 原子位置）题目 JSON 写进 task_prompt
Ground truth 数值 + 评分容差题目 JSON
Pseudopotential 家族（只有 GBRV USPP PBE）audit 禁联网，下载不了别的
QE 二进制版本（7.4）infra 决定
不许联网audit grep bash.log
不许后台跑 pw.x（& nohup disown）audit 检查
输出文件 ≥ 1024 B + JOB DONEaudit 检查

🟡 橙色：半硬编码 / 被框架软引导或硬截留

项状态
vc-relax 三阈值（forc/press/etot_conv_thr）v3 起 qe_input_lint pre-spawn hook强制改紧到 v3 默认
推荐的 ecutwfc / k-mesh 值v3 system prompt 推荐特定数（如 Si=60），但不强制——agent 写小了 bench 不拦，SCF 不收敛 / 算错时它自己才意识到
bash 命令白名单v3 起 bash_guard 拦截：黑名单 curl/pip/sudo，白名单只放 QE + coreutils + python
pseudo 路径必须用相对路径v3 prompt 强调 + pw.x 写错就找不到文件

🔵 蓝色：完全自由（agent 想怎么写就怎么写）

项影响
ecutwfc / ecutrho低于门槛结果错（见 §7.2）；高于门槛只是浪费时间
K_POINTS 网格同上
occupations / smearing / degauss金属 SCF 收敛性 + 数值噪声
conv_thr（SCF 收敛）准确度 vs 速度

nspin / starting_magnetization磁性体系必给
nbndbands 计算必须 > occupied
nstep / 优化算法 / BZ path / dos.x 区间各种细节都是 agent 自己拍
流程顺序（先 SCF 还是 NSCF）agent 自己排
失败后 retry 还是 incomplete submitagent 自己决定

宏观判断

真正"写死"的核心是：

  1. 题目 + 评分（bench 的产品核心）
  2. 不许联网 + 必须用 pseudos/ 里的赝势（沙盒安全）
  3. audit invariants（防作弊：JOB DONE / 文件大小 / 路径在 workdir）
  4. vc-relax 三阈值（v3 主动剥夺，因为放松必然空间群降级）

其他全是 agent 自由发挥。system prompt 的存在就是"诱导"agent 用合理值—— 写差了 bench 不拦，agent 自己 SCF 不收敛 / 算错 / 扣分。

如果未来想真正测 agent 的"思考能力"，可以反过来——把 system prompt 里的推荐值删掉， 让 agent 真的得做收敛测试。那时候 v5 之后的"自动收敛测试子 agent"才真有价值。
