§2 DFT 是什么、能算什么

DFT = Density Functional Theory（密度泛函理论），一句话讲清楚：

DFT 是一种从第一性原理（量子力学）出发，预测原子排好之后电子云怎么排的计算方法。 你给它一组原子的位置（"金刚石结构的 Si"），它给你回算：体系总能量、电子云分布、原子受力、 晶体光学和电子学性质（能带、态密度、磁性、介电常数...）。

更具体地，DFT 把多电子薛定谔方程（精确但跑不动）转换成一个等价的单电子问题 （Kohn-Sham 方程），代价是引入一个叫"交换关联泛函"的近似（见 §5 XC functional 卡片）。 "近似"是这个领域 60 年来争论的核心——LDA / PBE / SCAN / HSE 是不同的近似策略。

你可以把 DFT 想成一个超精确的物理模拟器：输入原子怎么排，输出"假如这些原子按薛定谔方程演化、达到稳态后会长啥样"。 跟分子动力学（MD）不同，DFT 算的是电子结构本身，不是原子怎么动；但有了电子的能量曲面， 原子的力就自然能算出来，进而能做结构优化、声子分析、热力学。

DFT 能算出来的物理量（不完全列表）

物理量研究员关心它干啥QE 工具
总能量 (total energy)比较两个结构哪个更稳定（最低能即最稳）pw.x scf
原子受力 (forces)结构优化的依据；非零 → 原子还想动pw.x scf, relax, vc-relax
应力张量 (stress)晶胞优化的依据；非零 → 晶胞还想变形pw.x vc-relax
电子能带 (band structure)判半导体 / 金属 / 绝缘体；能隙多大pw.x bands + bands.x

你 bench 里目前涵盖的是前 5 类——T1 关心总能量+力+应力（→ 优化好结构 → 量晶格常数 → 验证空间群）， T2 关心能带 + DOS。声子 / 磁性 / 高级泛函 / TDDFT 都还没涉及。

§3 Quantum ESPRESSO 是谁

Quantum ESPRESSO（简称 QE）是一个开源、平面波 + 赝势的 DFT 软件套件，主要由意大利 SISSA / CNR 团队维护，Fortran 写的，跑在 HPC 集群上。它是 DFT 软件生态里使用最广的几个之一：

软件商业 / 开源基组类型典型用途
Quantum ESPRESSO开源 (GPL)平面波 + 赝势周期晶体、能带、声子、磁性
VASP商业（贵）平面波 + PAW同上，工业界主流
ABINIT开源 (GPL)平面波 + 赝势同上，欧洲学术圈
GPAW开源 (GPL)real-space grid / PAW跟 ASE 集成好
FHI-aims商业 / 学术免费numeric atomic orbitals分子 + 晶体、全电子
Gaussian / NWChem商业 / 开源高斯轨道分子化学，不是晶体首选

本项目用 QE，几个具体原因：开源、HPC 友好、平面波适合周期晶体、社区教程多、有 GPU 加速版（NVIDIA HPC SDK）。

你只需要关心 QE 套件里这几个二进制：

§5 关键概念入门集（12 张卡片）

不用一次记住所有术语——后面 §6-§9 用到哪个你回来查哪个。每张卡片底部都有"在 Si 例子里它长啥样"的回扣。

💡 1/12

Plane-wave basis · ecutwfc · ecutrho

QE 把电子的波函数展开成平面波（傅里叶级数）的叠加。平面波数量由ecutwfc（波函数截断能，单位 Ry）控制， 越大越精确越慢。ecutrho 是电荷密度用的截断（一般是 ecutwfc 的 4×，USPP / PAW 要 8-12×）。

Si 用 ecutwfc = 60、ecutrho = 480（8×）。如果你换成 norm-conserving pseudo， 4× 就够（240）；用 USPP 不到 8× 会精度差。

💡 2/12

Pseudopotential（赝势）

原子内层电子离原子核近、几乎不参与化学键合，但数值上波函数振荡剧烈，会让平面波数量爆炸。 解决方案：把核 + 内层电子用一个"赝势"打包替换掉，只对外层（价）电子算。

三种主流赝势：NC (norm-conserving) — 最严格、要 ecutwfc 高； USPP (ultrasoft) — 更省截断、需要 8-12× ecutrho； PAW (projector augmented wave) — 兼顾精度和效率，VASP 主流。

本项目用 GBRV USPP（PBE 版）——开源、有完整周期表覆盖、社区基准成熟。 Si 的赝势文件叫 si_pbe_v1.uspp.F.UPF。

💡 3/12

Exchange-Correlation Functional（XC 泛函）

Kohn-Sham 方程里有个"交换关联泛函"项是没法精确写出的，必须用近似。 近似有几个层级（"Jacob's ladder"）：LDA（局域密度）→ GGA（PBE 最常用）→ meta-GGA（SCAN）→ hybrid（HSE）→ random phase approximation。 越高级越准但越慢（HSE 比 PBE 慢 10-100 倍）。

本项目默认 PBE（写在 pseudo 的 _pbe_ 后缀里）。PBE 对半导体能隙 预测偏小 30%-50%；想精确算能隙得用 HSE。你 bench 的 ground truth 用 PBE 算的所以可以直接比。

💡 4/12

k-mesh · Brillouin Zone · Monkhorst-Pack

周期晶体的电子态由布里渊区（Brillouin zone, BZ，倒空间的原胞）里的 k-vector 标记。 积分 BZ 用有限 k-mesh 离散化近似——这就是 K_POINTS automatic 8 8 8 里的 8×8×8 网格 （Monkhorst-Pack 是网格生成方案）。

金属需要密 k-mesh（16×16×16+）因为 Fermi 面要细化；绝缘体粗一点（6×6×6）够了； 单胞大的（4+ 原子）k-mesh 可以稀疏。k-mesh × 晶格常数 ≈ 常数 是个经验法则。

Si bulk 用 8×8×8 ok；nscf for DOS 一般再加密到 16×16×16；nscf for bands 是路径而不是网格（见 §8）。

💡 5/12

SCF 自洽循环 · mixing_beta

Kohn-Sham 方程的解是个鸡生蛋——电子密度依赖势场、势场又依赖电子密度。 解法：自洽迭代（Self-Consistent Field, SCF）：猜个初始密度 → 算势场 → 解方程 → 得新密度 → 跟旧密度混合（mixing_beta 控制比例，默认 0.7）→ 重复，直到密度收敛（conv_thr）。

"不收敛"是 DFT 第一大常见错误：mixing_beta 太大会震荡，太小会慢； 金属需要小 mixing_beta + smearing；磁性体系需要好的 starting_magnetization。

Si bulk 4 次 SCF 就收敛（绝缘体好脾气）。但 v3 lint hook 会在 SCF 失败时记录到 lint.log 提示重试， v5 直接调子 agent convergence_fixer 处理这种情况。

💡 6/12

Smearing（占据数光滑化）

原本电子按 Fermi-Dirac 分布占据能级——零温下能级要么 0% 要么 100% 占据， 不连续会让金属的 Fermi 能积分难收敛。Smearing 就是人为加点温度（degauss=0.01 Ry） 让占据数变光滑——SCF 才能收敛。

常用 smearing 类型：gaussian（最简单）、mv（Marzari-Vanderbilt 冷涂法 - 推荐金属）、 fd（真 Fermi-Dirac）、mp（Methfessel-Paxton）。

Si 是半导体本来不需要 smearing，但放 degauss=0.01 + 'gaussian' 也 OK，不会影响精度。 真正金属（Al, Cu）用 'mv' + degauss=0.02。

💡 7/12

vc-relax · 结构优化

vc-relax = variable-cell relaxation。同时优化原子位置（让 forces 趋零） 和晶胞（让 stress 趋零），直到达到收敛条件：forc_conv_thr（默认 1e-3 Ry/Bohr， 本项目严到 1e-4）和 press_conv_thr（kbar，本项目 0.05）。

另一种 relax 只优化原子位置不动晶胞，用于有衬底或界面体系。

T1 题目就是 vc-relax。v3 的 qe_input_lint 自动收紧阈值 → 防止松弛完了空间群被 pymatgen 降级。

💡 8/12

Brillouin Zone path · seekpath · k-label

能带图横轴是 BZ 里的一条高对称路径——Γ → X → W → L → Γ → K 之类。每个空间群有"标准 path"， 人类查表或用 seekpath（Python 库）自动给。每个端点叫 k-label（Γ / X / L / K / W 等）。

Si (fcc, 空间群 227) 的标准 path 用 seekpath 自动出来，一般是 Γ → X → U → K → Γ → L → W → X。 T2 评分会查 VBM / CBM 落在哪个 k-label 上——Si 是间接 gap（VBM at Γ, CBM not on path）。

💡 9/12

Band Gap · VBM · CBM · direct/indirect

VBM = Valence Band Maximum（价带顶 / 最高占据态）； CBM = Conduction Band Minimum（导带底 / 最低空态）； band gap = CBM − VBM 能量差。
如果 VBM 和 CBM 在同一个 k 点，叫 direct gap（光跃迁直接）； 不在同一 k 点叫 indirect gap（需要声子辅助才能跃迁）。

Si 是经典 indirect gap：VBM 在 Γ，CBM 在 BZ 内部 k = (0.85, 0, 0.85)（OFF_PATH——不在标准 high-symmetry path 上！）。 T2 评分要 agent 报对 vbm_k_label="GAMMA" + cbm_k_label="OFF_PATH"。

💡 10/12

Density of States · Fermi energy

DOS(E)：每能量 E 上有多少电子态——一条曲线。从 SCF 后的另一次 NSCF on dense k-mesh 算出来。
Fermi energy（e_fermi）：零温下电子的最高占据能量。绝缘体的 Fermi 落在 gap 中（DOS=0）， 金属的 Fermi 落在 DOS≠0 处。

T2 评分查 dos_e_fermi + 整条 dos_curve（在 Fermi 附近 ±5 eV 范围里跟 ground truth 做 L2 距离）。 Si 的 Fermi 是 6.25 eV（PBE 计算值）。

💡 11/12

空间群（Space Group） · pymatgen 怎么判

晶体有 230 个空间群，描述晶体的对称性。算空间群编号需要：晶胞 + 原子位置 + 对称容差（symprec，默认 0.01 Å）。

pymatgen 的 SpacegroupAnalyzer 是事实标准——给它 Atoms 对象， 返回空间群编号、点群、晶系（cubic / hexagonal / ...）。

Si 的空间群是 227 (Fd-3m)——金刚石结构。T1 评分查 a/b/c/α/β/γ + space_group_number。 **这里有个陷阱**：vc-relax 阈值太松，最终 cell 会偏离对称 ≥ 0.5°，pymatgen 在 symprec=0.01 下识别为低对称 （如三方 166 而不是立方 227），→ 直接扣 0.25 分。v3 的 lint hook 就是为这个出生的。

💡 12/12

QE input 文件结构

QE 的输入文件用Fortran namelist 格式。一个 pw.x 输入文件长这样：

&CONTROL
  calculation = 'scf'    ! 算啥：scf / nscf / bands / relax / vc-relax
  prefix = 'si'          ! 临时文件前缀
  outdir = './out'       ! 临时文件放哪
  pseudo_dir = './pseudos'
/
&SYSTEM
  ibrav = 0              ! 0 表示用下面的 CELL_PARAMETERS
  ecutwfc = 60           ! Ry
  nat = 2 ntyp = 1
/
&ELECTRONS
  conv_thr = 1.0d-8      ! SCF 收敛阈值
/
ATOMIC_SPECIES
  Si  28.086  si_pbe_v1.uspp.F.UPF
ATOMIC_POSITIONS crystal
  Si  0.00  0.00  0.00
  Si  0.25  0.25  0.25
CELL_PARAMETERS angstrom
  0.0  2.72 2.72
  2.72 0.0  2.72
  2.72 2.72 0.0
K_POINTS automatic
  8 8 8  0 0 0

四个必填 namelist（&CONTROL、&SYSTEM、&ELECTRONS、&CELL + 几个 card 块（ATOMIC_SPECIES、ATOMIC_POSITIONS、CELL_PARAMETERS、K_POINTS）。

注释符是 !（不是 #）；浮点用 1.0d-4（不是 1.0e-4，d 是 double）；布尔用 .true. / .false.。 这些小坑就是 LLM 第一次写 QE input 最容易翻的。v3 的 qe_tools.py 直接给函数封装这些细节。

§6 Si bulk 贯穿 · 准备阶段

从现在开始我们用 Si 单晶（金刚石结构）把全套流程跑一遍。Si 是 DFT 的 "Hello World"—— 半导体经典材料、空间群 227 (Fd-3m)、间接 gap、参数都背得滚瓜烂熟。

6.1 拿初始结构

研究员一般从 Materials Project 拿（Si 的 entry 是 mp-149）。 里面有实验测好的晶格常数 a = 5.43 Å，原子位置 Si in 8a Wyckoff position。 在 bench 里这个结构由 task_prompt 给好，agent 直接读。

6.3 跑、看收敛

跑命令很简单：pw.x -in scf.in > scf.out。Si bulk 8 核 CPU 约 30 秒跑完。 看 scf.out 关键几行（取自真实输出）：

     total energy              =     -19.19052547 Ry      ! 第 1 轮 SCF
     total energy              =     -19.19289405 Ry      ! 第 2 轮（在收敛）
     total energy              =     -19.19303975 Ry      ! 第 3 轮
     the Fermi energy is     6.4266 ev
!    total energy              =     -19.19308046 Ry      ! 第 4 轮（! 前缀=收敛后的最终值）
     convergence has been achieved in   4 iterations

4 次迭代就收敛了——绝缘体 SCF 一般 5-10 次。"convergence has been achieved" 这行没有意味着失败。

传统人类：先猜 ecutwfc，从低到高扫一遍（30 → 40 → 50 → 60 → 80），看总能差小于 1 mRy 就停——这就是 "收敛测试"。同样地扫 k-mesh：4×4×4 → 6×6×6 → 8×8×8。一个新材料这一步要 1-2 天。

Agent：bench 把 ecutwfc=60 / k=8×8×8 直接写进 v3 的 system prompt（"Si 类材料用这个值"）， 跳过收敛测试。这是有代价的——agent 不知道这个值对 W、Hf 等过渡金属是否够； v5 的下一步是加 "automatic convergence test" 子 agent，自己扫一遍。

§7 Si bulk vc-relax（对应 T1）

7.1 vc-relax 跟 SCF 的差异

SCF 是固定结构算电子态。vc-relax 是在SCF 外面套一个优化循环—— 每个 SCF 跑完后看 forces 和 stress，调整原子位置 + 晶胞，再跑下一轮 SCF。直到 force / stress 都低于阈值。

对应 input 的差异：&CONTROL 里 calculation = 'vc-relax'， 然后新增三个 namelist：

&IONS
   ion_dynamics     = 'bfgs'           ! 原子位置优化算法（BFGS 拟牛顿）
/
&CELL
   cell_dynamics    = 'bfgs'           ! 晶胞优化算法
   press_conv_thr   = 0.05            ! 压力收敛阈值 (kbar) — v3 lint 严的默认
/

! 同时在 &CONTROL 里加：
   etot_conv_thr    = 1e-05           ! 能量收敛阈值 (Ry)
   forc_conv_thr    = 0.0001          ! 力收敛阈值 (Ry/Bohr) — 严
   nstep            = 100             ! 最多 100 轮优化

三个阈值（etot_conv_thr / forc_conv_thr / press_conv_thr）就是 v3 lint hook 会自动收紧的那三个。 QE 默认值都松一个数量级，松弛完结构会偏离对称导致空间群降级。

7.2 跑完看输出

    已修：v3 的 qe_input_lint 在 bash 调 pw.x 前自动改 input 文件把三个阈值收紧到 v3 默认。
  2. 跑完忘了用 pymatgen 验证：agent 只看到 "JOB DONE" 就 submit，没看空间群是不是对——审计才发现降级。

    未修：v5 加 budget-aware termination 后能在 timeout 前提交"现有最好结果"。

§8 Si bulk SCF + bands（T2 第一段）

能带计算分两步——这是 QE 跟其他软件区别最大的地方之一。

8.1 Step A · SCF on uniform k-mesh（建电荷密度）

用 §6 的 scf.in（同 8×8×8 网格）跑一次，得到自洽的电子密度。 关键产物：out_scf/si.save/charge-density.dat（电荷密度二进制）。

8.2 Step B · NSCF along BZ path（在路径上求本征值）

"NSCF" = Non-Self-Consistent Field：固定上一步得到的电荷密度，不再自洽迭代， 只在新的 k 点列表上求本征值。这能省时间。

k-list 由 seekpath 自动给——对 fcc 空间群 227，得到一条 Γ → X → U → K → Γ → L → W → X 的路径，每段插值出 ~30 个 k 点，总共 ~240 个 k 点：

! bands_nscf.in 关键片段（取自 Si T2 真实运行）
&CONTROL
  calculation = 'bands'            ! 注意是 'bands' 不是 'nscf'——QE 的 quirk
  prefix = 'si'
  outdir = './out_scf'              ! 复用 SCF 的 out 目录（读 charge density）
/
&SYSTEM
  ibrav = 0
  ecutwfc = 60
  ecutrho = 480
  nbnd = 16                       ! 算多少个能带（默认只算占据态，bands 要多算空态）
/
&ELECTRONS
  conv_thr = 1.0d-8
/
...
K_POINTS crystal
  241                              ! 241 个 k 点（沿 BZ path）
  0.00000000  0.00000000  0.00000000  1.0
  0.01250000  0.00000000  0.01250000  1.0
  0.02500000  0.00000000  0.02500000  1.0
  ... (240 more)

nbnd 是这里特别要小心的：QE 默认只算占据态（Si 8 个价电子 / 原胞 = 4 个能带）， 但能带图需要看到 Fermi 以上的空态。一般 nbnd = 2 × occupied_bands + 几个 buffer—— Si 至少 8、稳妥 16。少了导带画不全，多了浪费 CPU。

8.3 Step C · 用 bands.x 把 NSCF 结果整理成 bands.dat

NSCF 结束后 out_scf/si.bands.dat 还是 QE 内部格式。bands.x 是后处理工具， 把 241 个 k 点 × 16 个能带的本征值整理成画图友好的格式。输入文件很简单：

&BANDS
  prefix = 'si'
  outdir = './out_scf'
  filband = 'bands.dat'             ! 输出文件名
  lsym = .false.                  ! 不做对称性 unfold
/

跑 bands.x < bands_pp.in > bands_pp.out，产出 bands.dat（每行：k_distance + 16 个能带能量）。 拿这个文件直接画图：横轴 k 路径、纵轴能量、16 条曲线就是能带。

8.4 拿能带数据找 VBM / CBM

对 Si（8 价电子，4 个占据能带）：

  1. 每个 k 点的能带按能量升序排，前 4 个是占据带（valence），后 12 个是空带（conduction）
  2. VBM = max over k 点 of 第 4 个能带能量 → 找到 VBM 在哪个 k 点（Si 是 Γ）
  3. CBM = min over k 点 of 第 5 个能带能量 → 找到 CBM 在哪个 k 点 （Si 在 BZ 内部约 (0.85, 0, 0.85)，不在 high-symmetry path 上 → label = OFF_PATH）
  4. Band gap = CBM − VBM = 0.62 eV（PBE 算出来，实验值 1.12 eV——PBE 低估）
  5. Gap type = indirect（VBM 和 CBM 不同 k 点）

T2 的 ground truth 字段（来自 M1_T2_Si_bands_dos.json）：

字段期望值容差
band_gap_ev0.621±0.1 eV
gap_typeindirectexact
vbm_k_labelGAMMAexact
cbm_k_labelOFF_PATHexact
传统人类：跑 bands → 在 python 里读 bands.dat → matplotlib 画图 → 肉眼找 VBM / CBM 的 k 位置。 熟练老手用 ASE-bandstructure 一行命令出图。

Agent 的脆弱点：
  1. NSCF 误用 'nscf' 而不是 'bands'：QE 里要做 band-structure NSCF 一定要写 calculation = 'bands'，写 'nscf' 算出来 k 点对不上。
    v5 的 MCP write_qe_input(calc="bands", ...) 强制 calc 字段必须是合法的几个值之一。
  2. nbnd 选少了：默认 occupied_only=4，agent 把这数写进去 → 导带就 0 条 → CBM 找不到。
    v3 system prompt 写了 "nbnd = 2× occupied + buffer"，v5 工具描述里也强调。
  3. VBM / CBM 的 k 标签算错：Si CBM 在内部 OFF_PATH 而不在 X / L 等标准点上—— agent 容易把"距离最近的标准 k 点"误报。
    没现成自动化——需要后处理脚本判 "最近 high-symmetry k 距离 < tolerance 才报 label，否则 OFF_PATH"。
  4. 忘了先跑 SCF，直接跑 NSCF：bands.x 读 out_scf/si.save/ 里的 charge density， 没 SCF 这步就没有。
    v3 system prompt 显式列出 5 阶段；v5 的 verify_run 强制检查 SCF 输出存在才放过。

§9 Si bulk DOS（T2 第二段）

态密度（DOS）需要的 k-mesh 跟 bands 完全不同——bands 是沿路径稀疏采样， DOS 是整个 BZ 均匀采样越密越准。所以又得跑一次 NSCF，参数不一样。

9.1 NSCF on dense k-mesh

! dos_nscf.in 关键差异
&CONTROL
  calculation = 'nscf'             ! 这次写 'nscf'（不是 'bands'）
  ...
/
&SYSTEM
  ...
  occupations = 'tetrahedra'       ! tetrahedra 法适合金属 DOS 积分；半导体也能用
  nbnd = 16
/
K_POINTS automatic
  24 24 24  0 0 0                ! 比 SCF 的 8×8×8 密 3 倍（24×24×24 / 8×8×8 ≈ 27 倍 k 点）

9.2 用 dos.x 后处理

&DOS
  prefix = 'si'
  outdir = './out_scf'
  fildos = 'dos.dat'               ! 输出文件名
  Emin = 1.19                     ! 能量积分范围下限 (eV，Fermi - 5)
  Emax = 11.19                    ! 上限 (Fermi + 5)
  DeltaE = 0.01                   ! 能量步长
/

跑 dos.x < dos.in > dos.out，产出 dos.dat（两列：能量 + DOS 值）。 顶部一行会有 Fermi energy（如 "# E (eV) dos (states/eV) ... # EFermi = 6.2538 eV"）。

T2 的 ground truth 字段：

字段期望值容差

传统人类：从 scf.out / dos.out 里 grep "Fermi energy" 一行就有 e_fermi。画图 / 跟实验对照看 DOS 形状是不是合理。

Agent 的脆弱点：
  1. DOS k-mesh 不够密：拿 8×8×8 直接跑 dos.x → 曲线锯齿严重 → L2 距离过大扣分。
    v3 system prompt 推荐 16-24× 密；v5 的 write_qe_input 检测 calc='nscf' 且用途是 DOS 时给建议值。
  2. 忘了跑 dos.x：只跑 nscf 就 submit，导致 dos.dat 不存在 → 评分系统找不到 → 0 分。 "忘了跑后处理"是真实出现的错误模式。
    v3 用 system prompt 反复强调 "5 阶段必须走完"；v5 的 verify_run 强制查文件存在。
  3. Fermi 能量读错位置：scf.out 和 dos.dat 各有一份 Fermi，应优先用 dos.dat 头部的（因为 dos.x 重新计算了一次）。agent 容易报错版本。
  4. Emin / Emax 设错：agent 不知道当前 Fermi 大致在哪 → Emin = 0 / Emax = 30 等 → 浪费时间且 dos.dat 太长。 v3 prompt 让 agent 先看 scf.out 拿到 Fermi 再设。

§13 进阶主题（点到为止）

你 bench 暂时不涉及这些，但实际研究 / 未来 bench 扩展时绕不开。每个主题给关键词，你想深挖时往这查。

Phonons（声子）
晶体的振动模式。QE 用 ph.x 算（密度泛函微扰理论 DFPT）。能给你声子色散、声速、热导、动力学稳定性 （虚频 = 结构不稳）。计算量大、k-mesh 更密。
DFT+U
过渡金属氧化物的 d / f 电子，纯 DFT 算出来太离域（band gap 偏小 / 磁矩不对）。加一个 Hubbard U 修正 （Hubbard_U(1) = 4.0）就能 fix。U 值靠经验或线性响应法定。
磁性 / Spin-polarized DFT
设 nspin = 2 + starting_magnetization。能算铁磁 / 反铁磁。 更复杂的 spin-orbit / non-collinear 用 nspin = 4。
Hybrid functionals (HSE)
PBE 混入一部分 Hartree-Fock 精确交换。半导体 band gap 准很多，但慢 10-100 倍。
热力学 / 相变
能量 + 声子 → 自由能；不同温度 / 压力下哪个相稳。要做的就是声子算多次 + Quasi-Harmonic Approximation。
TDDFT / 激发态
含时密度泛函，算光吸收谱。turbo_lanczos.x / tddfpt。准确度有限—— 高精度激发态要 GW + BSE（QE 配 BerkeleyGW 或 yambo）。
ML potentials
把 DFT 力训练成神经网络（NEP / DeepMD / MACE）。一次 DFT 计算 = 1000 个 MD step 的数据。 未来 agent 工作流可能不是"跑 DFT"，而是"跑 ML potential，DFT 只用作 ground truth"。

§14 给 agent 工程师的建议

读完这份文档你应该对"DFT agent 该自动化什么"有更清晰的判断了。我把这几个观点给你：

14.1 哪些环节适合 agent 自动化

  • 结构 → input 文件的代码生成：Fortran namelist 是机械翻译，没有创造性。v3 的 write_pw_input 已经做了； v5 的 write_qe_input MCP 工具进一步把它结构化。
  • 输出文件的 grep + parse：JOB DONE / 总能量 / Fermi / cell / 收敛标记都是字符串匹配。 v5 的 read_qe_output 用 window 函数（tail / errors / forces / scf_steps）取代盲打 grep，让 LLM 不需要重新发明搜索词。
  • SCF / vc-relax 收敛阈值的自动收紧：固定规则（v3 默认值就那三个），可以 lint。 不用让 LLM 决定，pre-spawn hook 直接改。
  • 提交前的程序级 sanity check：JOB DONE 在不在、文件大小够不够、空间群有没有降级—— 机械检查，比让 LLM 自查可靠。v5 verify gate 做这个。

14.2 哪些环节还需要"人"判断

  • 选 pseudo：是用 USPP / NC / PAW？哪个数据集（GBRV / PSlibrary / SSSP）？ 是否需要 semi-core 电子？这些问题没有死规则，靠经验。bench 目前规避——pseudo 路径已经写死。
  • 判断材料是否有磁性 / spin-orbit / DFT+U：3d/4f 元素的初值磁矩、Hubbard U 值—— 凭经验。bench 暂时只放无磁性体系绕开。
  • 定义 "正确" 的 k-mesh / ecutwfc：传统人类做收敛测试得到。bench 把这些值写死在 prompt 里， agent 跳过收敛测试——对未见过的元素有风险。
  • 当 SCF 怎么试都不收敛时，知道是放弃还是换 functional / 换初值： v5 子 agent 实现了 6 条 playbook 但更"创意性"的解决方案还是要人。

14.4 还能继续做的事

  1. 收敛测试自动化：让 agent 在跑大计算前先扫 ecutwfc / k-mesh 2 个点确认收敛。10 分钟成本，省后续大头。
  2. 用 ASE 当中间层：不让 LLM 直接拼 QE input，而是先生成 ASE Atoms + 配置字典，再让 ASE export 成 QE format。 减少 Fortran namelist 错误。
  3. 磁性检测：根据元素表自动判定 3d/4f 元素 → 是否需要 spin-polarized + DFT+U。一段简单查表逻辑。
  4. 把 baseline 跑分数据反哺 prompt：v5 的 experience 库已经做了 embedding 检索，但目前只是单向"借鉴"。 未来可以让 agent 把"哪些 prompt 在 Cd / WO3 上稳定 work"主动总结成新规则注入。
  5. 多 agent 协作：一个 agent 负责 vc-relax（结构优化）、一个负责 bands（电子结构）、一个负责验证。 v5 子 agent 是第一步但只对 SCF convergence 用了。
