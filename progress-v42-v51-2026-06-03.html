1 · 我们在做什么（30 秒背景）

我们在做一个用大模型自动跑 DFT 计算的 agent。通俗讲：DFT（密度泛函理论）是材料计算的主力方法，传统上要专家手写一堆 Quantum ESPRESSO（QE）输入、跑、看结果、反复调。我们让大模型把这套流程自动跑通——给它一道题（某个材料、要算什么），它自己写输入、跑 QE、读输出、检查、交卷。

有一个独立的 bench（测评系统）给每道题自动打分（0–1 分），对照标准答案算每个物理量对不对。所以"agent 强不强"是一个可量化、可对比的分数。下面所有数字都来自这个 bench。

2 · v4.2：通用成熟基线，标准集 27 题 0.918

v4.2 是两条轨道共同的起点，也是目前最成熟的通用版本。它把一道题拆成"规划 → 执行 → 验收"三段流水线，每段由一个大模型角色负责；验收不过就自动重试 / 重新规划。

图 1 · v4.2（也是 v4.3）的整体流程：两道前置功课 + 三个 LLM 角色接力 + retry / replan 反馈环。

通俗读法：开场两道功课（判金属/绝缘体、先预弛豫）由纯代码做；之后 planner 拆步、executor 执行、verifier 验收，不通过就自动重试 / 重新规划；执行环节外还有一层"输入自动纠错"兜底。

核心思想

把 DFT 知识写成"模型绕不过的代码硬约束"

老版本是把 DFT 经验写在提示词里，指望模型照做；v4.2 的关键升级是把这些经验变成代码层的硬规则——交卷前自动检查、自动纠正输入参数、独立验收清单。模型"想偷懒也偷不了"。这让成绩从"看模型心情"变成"稳定可控"。

v4.3 · 手写线的磁性特化版

v4.3 是把 v4.2 直接 fork 出来、专门为磁性 / 金属材料特化的版本。架构和 v4.2 完全一样（同样的"规划→执行→验收"三段式，见上图），区别在于针对磁性材料这类"难收敛、易判错"的体系，加了一组特化处理：

  • 磁序探针：自动试"铁磁 vs 反铁磁"两种磁排列，跑两个单点能量、选更低的那个——磁性材料算对的第一步关键。
  • 金属能带保证：跑完后检查金属费米面附近有没有假"缝"，有就自动加密 k 点重算，避免金属被误判成半导体（典型如 Ni）。
  • 一组磁性工程修复：强制自旋极化（nspin=2）、反铁磁原子标记归一化、把总态密度的两个自旋通道加起来、单条命令墙钟封顶防超时。
  • 顺手修了 v4.2 的边角：例如上一节 Cd 失分（0.445）——v4.3 把它的截断从 50 提到 80 Ry 后回到 0.948。

一句话：v4.3 = 把成熟的 v4.2 针对"磁性材料"这块硬骨头手动打磨过的生产版本；相对 v4.2 多了约 5500 行磁性 / 金属 / 墙钟相关的修复代码。

v5.1 · 全新 SDK 结构化工具底盘

另一条路不再让模型自由拼 shell 命令，而是换成官方 agent SDK 的结构化工具：模型按一组带类型的"按钮"操作（写 QE 输入 / 跑 pw.x / 读输出 …），更安全、更可控、更容易加新能力、更好维护。v5.1 = 这套新底盘 + 把 v4.2 的"材料知识层"搬回来当开场功课。最新一版还把 v4.3 的磁性处理层搬了回来（反铁磁原子标记归一、nspin=2 总态密度求和、起步磁矩自动播种——已并入上面 0.683 这批结果），并在补 v5.1 自己的金属能带保证（自旋通道挑选，开发中，见 §6）。

图 2 · v5.1 一道题的完整流转。紫 = 输入，黄 = 知识层，蓝 = 执行循环，橙 = 卡死断路，绿 = 子 agent / 结果，红 = 验收门。

通俗读法：知识层先给好物理默认值 → 模型在结构化工具循环里写输入 / 跑 pw.x / 开窗读输出 → 提交前物理验收门把关，不过就带反馈重放；旁挂卡死断路器 + SCF 修复子 agent。这次还把 DeepSeek 切到这条 SDK 路径，修了 4 个评审 bug，单测 167 → 336 全过。

怎么理解两条线：v4.2 / v4.3 是"手写循环"架构——成熟、单题质量高，但靠不断叠加修复代码维护；v5.1 是"结构化工具"架构——工程上更安全、更可维护、更易扩展，这轮在最新广集上鲁棒性已经反超（12 道零失败）。现阶段两条线各有所长、不急于判定谁取代谁：v4.3 是当前磁性生产主力，v5.1 在 de-risk。

5 · 需要和 DFT 专家杨博确认的问题（核实后 4 条）

上一轮把旧清单核查后收敛到 2 条（另 2 条已修，见本节末）。这一轮又核实补充了 3 条新发现的待确认问题——都对着当前 bench 代码 / GT 数据核过、并纠正了原清单的一处误记。现共 4 条。

问题 1VBM / CBM 的 k-label 打分有两类坑

(a) 极值落在非高对称 k 点：现在 gap_type 和"极值在高对称点"的 k-label 都能拿分了（MnF₂ 实测 vbm/cbm 全 1.0）；但极值落在非命名点时匹配仍失败、丢满分——实测 MnO（vbm=0 / cbm=1.0，丢 0.10）、NiO（vbm+cbm=0，丢 0.20）；MgSiO₃ 的标准答案价带顶就在 (0,0,0.3) 非命名点。（BN_hex / CaTiO₃ / 锂辉石 的 GT 在命名点 H/Z/Y₂，不属此类。）

(b) GT 本身就是 null（无解局）：Li₃PO₄ 的 GT vbm_k / cbm_k = null。打分器对 null 的处理是：agent 输出任何值 → 0，输出 null 也 → 0（null-vs-null 没给满分）。所以这 0.2 权重怎么都拿不到。（全集里只有 Li₃PO₄ 是 k-label 全 null；原清单里的 LiCoPO₄ 其实 GT 是 "S" / 坐标、可正常打分，不在此列。）

▸ 给杨博的问题：(a) 非命名点的极值，放宽 0.05 容差、还是按到最近命名点的方向给部分分？(b) GT 为 null 时，null-vs-null 该给满分、还是把该字段权重置零？现在是无解局。

问题 2绝缘体的 dos_e_fermi 参考系 + DOS 曲线打分

绝缘体的 dos_e_fermi 现在仍是把价带顶绝对能量按 ±0.1 eV 绝对容差比——BN_hex 有稳定 0.36 eV 的系统偏移（不是抖动）直接判 0；实测最新磁性集 MnO / MnF₂ 的 dos_e_fermi 也都 = 0。另外 DOS 曲线用面积重叠 F1 打分，对"形状对、协议略偏"太宽容（实测 dos_curve 普遍只 0.2–0.5）。

▸ 给杨博的问题：① 绝缘体的费米能 / 价带顶该用什么参考系比（绝对 vs 相对、或跳过绝对 VBM）？② DOS 曲线该不该换更严的形状度量（KL 散度 / 归一化互相关）？

问题 3磁矩字段的容差：绝对 0.5 µB 合理吗？

T1 的 total_magnetization 和 absolute_magnetization 现在用绝对容差 0.5 µB（误差 ≤0.5 满分，0.5→1.0 线性递减，>1.0 为 0），而且两者合计占 T1 权重 0.35（单项最大头）。绝对 0.5 对大磁矩材料偏松（NiO ~6 µB → 仅 ~8% 相对误差），对小磁矩材料极松（~1 µB → 50% 相对）。

▸ 给杨博的问题：0.5 µB 绝对容差合理吗？换成相对容差（随磁矩大小缩放）是否更公平、更有区分度？

问题 4NiO / CoO 这类强关联体系：PBE-金属 vs Mott-绝缘体（设计意图需说清）

bench 的 GT 是 PBE + nspin=2、不加 +U，所以 NiO（GT 带隙 0.001 eV = metal）、CoO（0.003 eV）在 GT 里是金属；但物理上它们是 Mott 绝缘体（带隙 ~4 eV）。这不是 bug——bench 测的是"能否复现 PBE+nspin=2 的计算"，不是"能否给物理正确答案"，两者在这类体系上是矛盾的。

▸ 给杨博的问题：确认 bench 的口径就是"复现 PBE 计算"（NiO = metal 的 GT 是 by design 正确）？若是，建议在文档里写明，并界定它对 solver 的指导意义——solver 在这类体系上拿不到"物理对错"的信号。

本节末 · 已修复、无需再讨论：① 金属判据 100× 不一致（原议题）——"多小算金属"阈值已从 1 meV 抬到 10 meV，并按 GT 的金属标志 gate（已并 main），近零带隙绝缘体不再连环判错；仅剩 Rb 类（20–43 meV）的小边角。② DOS 每化学式归一化——已修（GT 侧重新生成成同基准）。
