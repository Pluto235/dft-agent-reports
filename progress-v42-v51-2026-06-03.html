1 · 我们在做什么（30 秒背景）

我们在做一个用大模型自动跑 DFT 计算的 agent。通俗讲：DFT（密度泛函理论）是材料计算的主力方法，传统上要专家手写一堆 Quantum ESPRESSO（QE）输入、跑、看结果、反复调。我们让大模型把这套流程自动跑通——给它一道题（某个材料、要算什么），它自己写输入、跑 QE、读输出、检查、交卷。

有一个独立的 bench（测评系统）给每道题自动打分（0–1 分），对照标准答案算每个物理量对不对。所以"agent 强不强"是一个可量化、可对比的分数。下面所有数字都来自这个 bench。

2 · v4.2：通用成熟基线，标准集 27 题 0.918

v4.2 是两条轨道共同的起点，也是目前最成熟的通用版本。它把一道题拆成"规划 → 执行 → 验收"三段流水线，每段由一个大模型角色负责；验收不过就自动重试 / 重新规划。

图 1 · v4.2（也是 v4.3）的整体流程：两道前置功课 + 三个 LLM 角色接力 + retry / replan 反馈环。

通俗读法：开场两道功课（判金属/绝缘体、先预弛豫）由纯代码做；之后 planner 拆步、executor 执行、verifier 验收，不通过就自动重试 / 重新规划；执行环节外还有一层"输入自动纠错"兜底。

核心思想

把 DFT 知识写成"模型绕不过的代码硬约束"

老版本是把 DFT 经验写在提示词里，指望模型照做；v4.2 的关键升级是把这些经验变成代码层的硬规则——交卷前自动检查、自动纠正输入参数、独立验收清单。模型"想偷懒也偷不了"。这让成绩从"看模型心情"变成"稳定可控"。

v4.3 · 手写线的磁性特化版

v4.3 是把 v4.2 直接 fork 出来、专门为磁性 / 金属材料特化的版本。架构和 v4.2 完全一样（同样的"规划→执行→验收"三段式，见上图），区别在于针对磁性材料这类"难收敛、易判错"的体系，加了一组特化处理：

  • 磁序探针：自动试"铁磁 vs 反铁磁"两种磁排列，跑两个单点能量、选更低的那个——磁性材料算对的第一步关键。
  • 金属能带保证：跑完后检查金属费米面附近有没有假"缝"，有就自动加密 k 点重算，避免金属被误判成半导体（典型如 Ni）。
  • 一组磁性工程修复：强制自旋极化（nspin=2）、反铁磁原子标记归一化、把总态密度的两个自旋通道加起来、单条命令墙钟封顶防超时。
  • 顺手修了 v4.2 的边角：例如上一节 Cd 失分（0.445）——v4.3 把它的截断从 50 提到 80 Ry 后回到 0.948。

一句话：v4.3 = 把成熟的 v4.2 针对"磁性材料"这块硬骨头手动打磨过的生产版本；相对 v4.2 多了约 5500 行磁性 / 金属 / 墙钟相关的修复代码。

v5.1 · 全新 SDK 结构化工具底盘

另一条路不再让模型自由拼 shell 命令，而是换成官方 agent SDK 的结构化工具：模型按一组带类型的"按钮"操作（写 QE 输入 / 跑 pw.x / 读输出 …），更安全、更可控、更容易加新能力、更好维护。v5.1 = 这套新底盘 + 把 v4.2 的"材料知识层"搬回来当开场功课。

图 2 · v5.1 一道题的完整流转。紫 = 输入，黄 = 知识层，蓝 = 执行循环，橙 = 卡死断路，绿 = 子 agent / 结果，红 = 验收门。

通俗读法：知识层先给好物理默认值 → 模型在结构化工具循环里写输入 / 跑 pw.x / 开窗读输出 → 提交前物理验收门把关，不过就带反馈重放；旁挂卡死断路器 + SCF 修复子 agent。这次还把 DeepSeek 切到这条 SDK 路径，修了 4 个评审 bug，单测 167 → 336 全过。

怎么理解两条线：v4.2 / v4.3 是"手写循环"架构——成熟、单题质量高，但靠不断叠加修复代码维护；v5.1 是"结构化工具"架构——工程上更安全、更可维护、更易扩展，这轮在最新广集上鲁棒性已经反超（12 道零失败）。现阶段两条线各有所长、不急于判定谁取代谁：v4.3 是当前磁性生产主力，v5.1 在 de-risk。

两条线各自的进展：v5.1 用修过的 solver 把鲁棒性拉上来了——这轮 12 道零失败、总分反超，并修掉了 MnO 这类自旋 bug；v4.3 则在单题质量上仍然领先（完成的题均分更高），短板是更大的磁性体系容易超时。谁赢取决于卷子和口径，现阶段不急于判定——两条线都在往前走。

5 · 需要和 DFT 专家杨博确认的问题（核查后收敛到 2 条）

05-25 那版列了 4 条；这次重新核过当前 bench 代码 + 实测最新分数，其中 2 条已经修掉（见本节末），真正还开放、需要杨博拍板的收敛到下面 2 条。

问题 1极值落在非高对称 k 点时，VBM / CBM 的 k-label 仍拿不到分

好消息：现在 gap_type、以及"极值在高对称点"的 k-label 都能拿分了。残留：当价带顶 / 导带底落在非高对称 k 点（偏离能带路径上的命名点）时，匹配仍失败、丢满分。当前 scorer 实测（最新磁性集）：

  • MnO：vbm_k = 0 / cbm_k = 1.0 → 丢 0.10
  • NiO：vbm_k = 0 / cbm_k = 0 → 丢 0.20
  • 对照 MnF₂：vbm_k / cbm_k 都 = 1.0（极值在高对称点，能拿分）→ 说明问题只在"极值离高对称点"时

▸ 给杨博的问题：对"极值不在高对称点"的绝缘体，k-label 怎么判才公平？放宽 0.05 的匹配容差，还是允许按到最近命名点的方向给部分分？（标准集的 MgSiO₃ / CaTiO₃ / LiAlSi₂O₆ / BN_hex 疑似同类，待用新 scorer 复核。）

问题 2绝缘体的 dos_e_fermi 参考系 + DOS 曲线打分

绝缘体的 dos_e_fermi 现在仍是把价带顶绝对能量按 ±0.1 eV 绝对容差比——BN_hex 有稳定 0.36 eV 的系统偏移（不是抖动）直接判 0；实测最新磁性集 MnO / MnF₂ 的 dos_e_fermi 也都 = 0。另外 DOS 曲线用面积重叠 F1 打分，对"形状对、协议略偏"太宽容（实测 dos_curve 普遍只 0.2–0.4）。

▸ 给杨博的问题：① 绝缘体的费米能 / 价带顶该用什么参考系比（绝对 vs 相对、或跳过绝对 VBM）？② DOS 曲线该不该换更严的形状度量（KL 散度 / 归一化互相关）？

本节末 · 已修复、无需再讨论：① 金属判据 100× 不一致（原议题）——"多小算金属"阈值已从 1 meV 抬到 10 meV，并按 GT 的金属标志 gate（已并 main），近零带隙绝缘体不再连环判错；仅剩 Rb 类（20–43 meV）的小边角。② DOS 每化学式归一化——已修（GT 侧重新生成成同基准）。
