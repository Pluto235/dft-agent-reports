1 · 我们在做什么（30 秒背景）

我们在做一个用大模型自动跑 DFT 计算的 agent。通俗讲：DFT（密度泛函理论）是材料计算的主力方法，传统上要专家手写一堆 Quantum ESPRESSO（QE）输入、跑、看结果、反复调。我们让大模型把这套流程自动跑通——给它一道题（某个材料、要算什么），它自己写输入、跑 QE、读输出、检查、交卷。

有一个独立的 bench（测评系统）给每道题自动打分（0–1 分），对照标准答案算每个物理量对不对。所以"agent 强不强"是一个可量化、可对比的分数。下面所有数字都来自这个 bench。

2 · v4.2：成熟基线，标准集 27 题 0.918

架构（一句话 + 一张图）

v4.2 把一道题拆成"规划 → 执行 → 验收"三段流水线，每段由一个大模型角色负责；验收不过就自动重试 / 重新规划。

题目+ 两道开场功课
→
Planner把题拆成步骤
→
Executor写输入 · 跑 QE · 读输出
→
Verifier逐步验收
→
结果交卷打分

验收失败 → 自动重试 / 重新规划。两道"开场功课"（判金属还是绝缘体、先把结构预弛豫一下）由纯代码完成，不劳烦模型。

核心思想

把 DFT 知识写成"模型绕不过的代码硬约束"

老版本（v1）是把 DFT 经验写在提示词里，指望模型照做；v4.2 的关键升级是把这些经验变成代码层的硬规则——比如交卷前自动检查、自动纠正输入参数、独立验收清单。模型"想偷懒也偷不了"。这让成绩从"看模型心情"变成"稳定可控"。

3 · v5.1：新底盘 —— 为什么换、更新了什么

30 秒理清版本：我们有两条并行实现线——手写线（v1 → … → v4.2 → v4.3，自己写的循环，能跑任何模型）和新 SDK 线（v5 → v5.1，基于官方 agent SDK 的结构化工具）。v4.2 / v4.3 是手写线的成熟版本，v5.1 是新线。后面 M3 的对照，就是拿新线 v5.1 和手写线当前头 v4.3 跑同一个模型做 A/B。

为什么换底盘

v4.2 物理很强，但工程上"脆"——模型直接拼 shell 命令操作，容易出幺蛾子、不好维护。v5 改用官方 agent SDK 的结构化工具底盘：模型不再自由打字敲命令，而是按一组带类型的"按钮"操作，更安全、更可控、更容易加新能力。v5.1 = 这套新底盘 + 把 v4.2 的"材料知识层"搬回来，争取"新底盘的工程 + 老版本的物理"。

v4.2 知识层preflight：材料知识 + 判金属/绝缘体 → 好的物理默认值
→
结构化工具
执行循环写输入 · 跑 pw.x · 开窗读输出
→
提交前
物理验收 gate不过就打回重做
→
结果

执行循环旁挂着两个"安全件"：卡死自动断路器、SCF 不收敛时交给一个专门的修复子 agent（干净脑子、按故障树修）。外框 = v5 的 SDK 底盘。

v5.1 更新了什么（相比手写线）

  • 结构化工具替代裸命令：模型按"写 QE 输入 / 跑 pw.x / 读输出"等带类型的按钮操作，更安全、更难出错。
  • 读输出"开窗"：只给模型看输出的关键片段（报错、受力、费米能…），不灌几千行——省上下文、少幻觉。
  • 卡死断路器：纯代码监控，发现模型原地打转（重复同一操作、长时间没进展）就自动打断并提示换招。
  • SCF 修复子 agent：自洽算不收敛时，交给一个"没看过失败现场"的专门子 agent 重新诊断。
  • 提交前物理验收 gate：模型说"做完了"不算数，程序再做一遍物理检查（收敛没、空间群保住没、能带文件能不能解析），不过就打回重做。
  • DeepSeek 也切到 SDK 路径：这次让 DeepSeek 也走上面这套结构化底盘（之前走裸命令）。
  • 修了 4 个评审 bug（C1–C4）：把 v5 里两处"物理验收被架空"的地方接回了线，并修了"跑挂了却被当成功"等问题。单元测试 167 → 336 全过。

总分 v5.1 暂时落后，但拆开看方向是好的：

材料v4.3v5.1占回归根因

为什么说是 bug 不是能力

QE 的物理输出本身是完整、正确的

Cr：模型其实把正确的高对称 k-路径能带算出来了，但交卷时抓错了文件——交上去的是稠密 DOS 网格，打分器找不到高对称点，4 个能带字段全判空。MnO：自旋极化体系的能带文件只写了一半（缺自旋向下通道），打分器对不上 → 带隙判空。两个都是 SDK 新路径"文件流转"还没打磨好，不是算错物理。

修复后的反推（正面落点）：把 Cr、MnO 恢复到正常水平、Fe₂O₃ 正常交卷，v5.1 的 T2 约回到 0.48，和 v4.3 的 0.518 基本持平；叠加 T1 的领先，整体 ≈ 打平。也就是说：新底盘在"还很早、IO 还没打磨"的阶段，已经追平了打磨成熟的老版本——这正是我们认为这条路线值得继续投的原因。M3 报告本身的结论也是"修这几个 bug 重跑，而不是退回老版本"。

另外有一个两个版本都一样的"天花板"：所有绝缘体的能带类型 / VBM / CBM 位置现在一律拿不到分。这不是某个版本的问题，而是 bench 打分口径的问题——正是下一节要找杨博敲定的。

5 · 需要和 DFT 专家杨博确认的问题（4 条）

下面 4 条是我们已经定位到的卡点，但属于"打分口径 / 物理判据"层面，需要杨博拍板。它们也正好解释了为什么 T2 有天花板、而且对两个版本都一样——不是 agent 算错。

问题 1"算出来几乎是零带隙"的金属判据，前后差 100 倍

同一个"带隙接近零"的物理事实，bench 用了两套差 100× 的标准：带隙数值容忍 ±0.1 eV（100 meV），但"是不是金属"的分类判定要 < 1 meV 才算金属。结果 Ge（算出 23 meV）、Rb（20–43 meV）这种近零带隙材料，带隙值判对、但金属/方向/k 点三个分类字段连环判错。

▸ 为什么重要：这类失分不是我们算错，是判据自相矛盾。需要杨博定一个合理的"多小算金属"口径。

问题 2绝缘体的 VBM / CBM 位置拿不到分 → T2 被封顶

所有真有带隙的绝缘体，能带类型 / 价带顶位置 / 导带底位置三个字段现在一律拿不到分（占每题 0.35 权重），等于绝缘体 T2 被封顶在 ~0.65。两个版本都一样。

▸ 为什么重要：这是 T2 分数最大的系统性天花板。需要 bench 侧加一个"把能带极值映射回高对称 k 点标签"的能力，并和杨博确认 k 点匹配口径是不是过严。

问题 3态密度（DOS）的归一化基准

标准答案的 DOS 是"每化学式"归一，QE 默认输出是"每原胞"，对 BN 这种原胞 = 2 个化学式的材料，我们的曲线天然高 2 倍。

▸ 为什么重要：是个一行就能修的口径问题，但要先和杨博确认归一化基准从哪取，避免冤枉失分。

问题 4DOS 费米能 / 曲线的参考系与打分方式

绝缘体的"费米能"字段实际比的是价带顶绝对能量，BN 出现稳定 0.36 eV 的系统偏移（不是抖动），更像"能量零点参考系"没对齐；而 DOS 曲线用面积重叠打分，对"形状对、协议错"太宽容。

▸ 为什么重要：需要杨博定费米能/价带顶的参考系，以及 DOS 曲线该不该换更严的形状度量。

6 · 短期改进规划（未来 1–2 周）

优先级从"直接把 v5.1 拉回持平/领先"的修复排起：

P0能带文件提交校验：要求交上去的能带文件点数 ≈ k-路径长度，否则打回重做。预期收回 Cr 类 ~+0.84/题。
P0自旋极化能带后处理覆盖双通道：写/解析时校验行数，自旋向上向下都要齐。预期收回 MnO 类 ~+0.33/题。
P1完成度兜底：产物齐全且预算还够时，强制走"验收 → 交卷"，并对 SDK 流中断自动重试。救回 Fe₂O₃，省 ~10 小时算力。
P2把物理纪律写进 SDK 工具代码：三条经验（起步 ecut 别拉太满、k 网格按晶轴各向异性、墙钟够用即收）目前只在老 prompt 里；DeepSeek 切到 SDK 后读不到，要"固化进代码"而不是靠提示词。
下一步v5.1 重跑标准集 27 题：现在 v5.1 只在 M3（磁性 6 题）上比过 v4.3，需要在标准集上和 v4.2 做一次同口径干净对比，才能下"v5.1 是否全面追平/超越"的结论。
bench 侧和杨博敲定打分口径：加 VBM/CBM 的 k 点标签映射、复核第 5 节那 4 条口径。利好两个版本。

一句话收尾：v4.2 已经把"能不能做对"证明了（0.918）；v5.1 把"能不能做得更稳、更可扩展"的底盘搭好了，现在卡在几个明确、可修的工程 bug 上。修完这几个、补上标准集对比、和杨博敲定打分口径，v5.1 就能从"打平"走向稳定领先。
