启动 server 的标准流程

每次起服务前 activate 自己的 runtime（"谁起服务谁写"）：

cd /srv/shared/science/web
cp ~/.web-runtime/env .env
cp ~/.web-runtime/config.toml config.toml
bash start.sh
# server 默认起在 localhost:8765
.env 和 config.toml 在共享目录里会被对方写过来覆盖——这是设计本意。 两人不要同时起服务（端口 8765 也会冲突）。约定好谁在用即可。

出问题怎么办

  • git 报 dubious ownership → 检查 safe.directory 是否加了对应路径
  • web 起来但读不到 runs → 检查 config.toml 里 bench_dir / demo_runs_dir 是否指对
  • weilin 写共享目录里某个文件 permission denied → 多半是该文件没有默认 ACL（shijie 修过的目录都有；如果撞到例外，setfacl -m u::rwx,g::rwx,o::rwx file 一下）
  • token 不工作 → 看 /srv/shared/science/web/tokens.yaml 是否包含你的条目；server 重启会重新 lifespan-load 它
  • 其它 → 找 shijie
