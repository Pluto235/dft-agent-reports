6. 给 DFT 合作者的两个讨论议题

议题 1 · 三类"难物理"任务的失分到底是 solver 可救还是 PBE 已尽力？

从本次 batch 里挑出 3 个分数 0.16 – 0.89 的物理上确实复杂的体系， 请合作者帮判断：是 solver 端的能带/极值搜索逻辑该升级，还是 PBE + 当前 GBRV pseudo + 当前 k-grid 已经到顶， 亦或 bench GT 的设定本身需要调整。

体系v4.2 表现问题
CoSb3 skutterudite第 1 跑 0.162（k-grid 稀 → 极值落非 Γ）；第 2 跑 0.979 这种"窄带隙 + 极值贴 Γ"的拓扑体系（半金属 / 候选热电材料），solver playbook 里应该硬编码"k-grid lower bound" 还是"非自适应做不了，必须人工标"？
MgSiO3 bridgmanite（钙钛矿）0.893；VBM 落 (0, 0.35, 0) 而 GT (0, 0, 0.30) 地幔钙钛矿这类近简并多极值的 valence top，是不是 GT 选了一个、solver 选了另一个但物理上等价？还是 solver 真的漏了 (0,0,0.30) 那个 k 点？
C diamond第 1 跑 0.348（报 Γ-Γ 直接带隙 5.58 eV）；第 2 跑 0.857 diamond 是 indirect gap (Γ → 0.85·X)，但 5.58 eV 也是 PBE 给 Γ 直接跃迁的合理值。 solver 应不应该被要求"看到一个直接带隙后必须再扫一遍非高对称点确认"？这是哪种材料必须做、哪种可以省的？

愿景：如果 DFT 同行能给一份"必须做完整 band path 扫描的材料类别清单" （如"任何含 d / f 电子的过渡金属化合物"、"任何 indirect gap 半导体"）， 我们就能把它编码进 v4.2 的 verifier 层，作为 §6 议题 2 的物理 sanity check 的一部分。

议题 2 · 给 v4.2 verifier 加哪些"物理常识 sanity check"？

2026-05-18 v4.2 架构评审报告 §2 已经提过： 现在 verifier 只检 audit_flags（QE 跑了？路径在 workdir？文件非空？），不检物理合理性。 经典反例是 GaN——已知半导体，v4.1 把它当 metal 抽出来，verifier 默默放行，最终 0.05。 本次 batch 里也有几个值得作为新 sanity check 的种子样本：

想请合作者拍板的优先级问题：以上 5 条 sanity check 假设全做出来， 您愿意接受 verifier 的假阴率（误判为不合理）有多高？ 我们目前的取舍是"宁可漏过也别误杀"（FIX-17 的 default-pass 哲学），但代价是 GaN 这种翻车不会被自动捕获—— 如果可以接受 ~10 % 的假阴率，前面 1-4 条都可以做出来；如果要 0 假阴，可能只有第 1 条值得上。

7. solver-team 改进 backlog（dev-facing）

本次 batch 暴露的 6 个工程问题

  1. Rb orchestrator 的 TypeError: unhashable type: 'slice' （/srv/shared/science/solver/v4.2/solve.py:383 附近）—— 让一道本来 QE 已经算完的题白挂了两次。修。优先级 P0。
  2. vc_relax 无 early-exit 机制 —— CsPbI3 和 WO3 都拿到了 1.000 满分，但花了 5500 + 5800 s（vs M1 vc_relax 一般 < 500 s）。 这是 perovskite / TM-oxide 收敛慢的固有 cost，但我们应该在 verifier 判定 "force / stress 已 below threshold + 空间群 / 晶格漂移已收敛" 时早出， 而不是死循环到 SCF iteration 上限。预估能省 30-50 % 的 T1 wall。
  3. Cd c-axis 4 % 偏差 —— 晶格角度和空间群都对，纯数值偏。怀疑是 occupations 矩阵在 alkaline / post-transition 金属边界 case 上选错了 smearing； 需要复现一次同参数能否手动跑出 GT 数值。
  4. k-label resolver 丢 kz（BN_hex VBM 标 K 而不是 H） —— hex BZ 里 K (1/3,1/3,0) 和 H (1/3,1/3,1/2) in-plane 一样，命名时需要带上 kz 判别。 新的 layered-vdw playbook 已经文档化但 labeler 没接住。
  5. off-symmetry k 点的 labeling 策略 —— MgSiO3 / CaTiO3 / Spodumene 都是 VBM/CBM 落到非高对称点，被 labeler 标成 fractional coords。 和 议题 §6 配合，先解决 labeler 一致性（同一 k 点 GT 和 solver 输出标法要可比），再讨论是否要 dense submesh refine。
  6. bench_fallback 路径不写 QE 仪表化字段 —— DB 里 4 个 run (Rb 3rd / CoSb3 2nd / CsPbI3 / WO3) 的 qe_call_count 和 qe_wall_sec 全 NULL。 要么在 harness 层补回填，要么把这 4 个字段从 NULLABLE 改成必填 + 强制 callsite 写入。 对 cost / 时间分析的可靠性有影响。
