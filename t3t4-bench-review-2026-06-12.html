1 · T3/T4 打分模式的修改 BENCH 已做·已 merge

1.1 新的"分级收紧曲线" graded_relative

和 DFT 专家定的口径：3% 满分 / 3–10% 线性递减 / ≥10% 零分。落地成对每个字段自动缩放的通用曲线 —— 把每个字段现有的 tolerance 重新定义为"零分线"（旧 relative 把它当"满分线"，所以新规则整体严约 3.3×）：

# /srv/shared/science/bench/harness/lib/scoring.py
_FULL_MARK_RATIO = 0.3

def _graded_score(err, tol, partial):
    if tol <= 0:
        return 1.0 if err <= 0 else 0.0
    full = _FULL_MARK_RATIO * tol      # 满分线 = 0.3·tol
    if err <= full:
        return 1.0
    if not partial or err >= tol:       # 零分线 = tol（硬归零，无旧版 [tol,2tol] 尾巴）
        return 0.0
    return max(0.0, min(1.0, 1.0 - (err - full) / (tol - full)))   # 中间线性

0.3 正好复现专家对 10% 字段的"3% 满分 / 10% 零分"，同时让每个字段按自己的 tolerance 自动缩放。 同一条曲线被 graded_relative / matrix_frobenius_relative / eos_curve_rmse / stress_curve_rmse 共用。

1.3 Cij：从原始 Frobenius 改成"特征值比较"（旋转不变）

问题 旧 matrix_frobenius_relative 逐元素比 6×6 Voigt 矩阵，对取向 / Voigt 约定差异敏感， 而这不是 solver 质量问题。实测 Si 一例：原始 Frobenius rel = 14%，但旋转不变量基本一致 → 44 个里 21 个被误杀（false-negative）。

改法（matrix_eigenvalue_relative）：① 对称化 (C+Cᵀ)/2；② 取本征值（Kelvin moduli，旋转不变量）； ③ 排序后比相对差 rel = ‖sort(eig_ext) − sort(eig_gt)‖ / ‖eig_gt‖；④ 套 _graded_score，tol = 0.08。 同一个 Si 例子，eigenvalue rel = 0.16%，正确判过。物理含义见 §4-F，需你确认。

2 · Prompt 写得太死 — 一个例子，请审核 BENCH 请定尺度

核心顾虑 现在的 prompt 几乎是把完整 recipe 抄给 agent —— 体积网格、BM3 解析式、scipy 方法、初值、单位系数、 应变模式 / 幅度、VRH 公式全给了。这等于在测"agent 能不能照着完整说明书把流程在 QE 里跑通"， 而不是"agent 懂不懂怎么算 EOS / 弹性"。这正是 v1 baseline 都能拿高分的根因之一。

下面是 T3 的完整 prompt 模板（schema/prompts/eos.txt），原文贴出供你判断处方化程度：

For {formula} ({structure_type}, space group {space_group}, {atoms_per_primitive_cell}
atoms per primitive cell), compute the equation of state (EOS) by Birch–Murnaghan
3rd-order (BM3) fit. The initial structure is the equilibrium configuration from a
prior vc-relax; use it as your V0 reference.

Workflow:
1. Generate 7 volume points by scaling the equilibrium cell volume by
   ratio ∈ {0.94, 0.96, 0.98, 1.00, 1.02, 1.04, 1.06} (uniform isotropic scaling
   of CELL_PARAMETERS).
2. For each volume, run a fixed-cell ionic relaxation with calculation='relax'
   (ions move, cell fixed). Use forc_conv_thr ≤ 1e-4 Ry/Bohr and
   etot_conv_thr ≤ 1e-5 Ry.
3. Take the converged total energy from each volume's pw.out ...
4. Fit the 7 (V, E) pairs to the BM3 equation of state:
     E(V) = E0 + (9·V0·B0/16) · [ (η-1)³·B0' + (η-1)²·(6 - 4η) ],  η = (V0/V)^(2/3)
   using scipy.optimize.curve_fit (Levenberg–Marquardt, no bounds).
   Initial guess: V0 = V_min(E), B0 = 0.6 eV/Å³, B0' = 4.0, E0 = E_min.
5. Convert B0 to GPa via × 160.21766208.
6. Report the three fit parameters as numbers (no units in the value).

Pick adequate ecutwfc, k-mesh, and SMP settings for production-quality energies.
Use the same numerical settings across all 7 volumes ...

Your task_outputs MUST include this key (path relative to your workdir):
- eos_points_csv: a CSV with header ratio,volume_ang3,energy_ry,pressure_kbar
  and 7 rows, values parsed from each pw.out.
[+ initial_structure_block / pseudopotential_block / advisory_notes / execution hint]

注意：连 BM3 解析式、scipy 的 LM 方法、初值、单位换算系数都给了。agent 唯一要"自己决定"的只剩 ecutwfc / k-mesh / SMP —— 而这几项又被"7 点保持一致"进一步约束。 T4 的 elastic_stiffness.txt 更甚：6 个 Voigt 应变模式、4 个幅度、应变张量构造、lstsq 拟合、对称化、 Voigt / Reuss / Hill 三套平均公式全部明写。

需要你定尺度：
  • 保持处方化 = 测"执行可靠性"（能不能不出错地把固定流程跑完）；
  • 放松成只给目标（"算 BM3 EOS，报 V0/B0/B0'"）= 测"方法学判断"（知不知道该取几点、用什么拟合、怎么保证一致性）；
  • 也可分档：M1 给 recipe、M2/M3 只给目标。

来源：bench/schema/prompts/{eos,elastic_stiffness}.txt

4 · Bench 侧仍需确认的问题（重点 T3/T4）BENCH 请决策

按优先级排序，A–B 最该先拍板。

A.【最关键】Prompt 处方化程度 — 测执行还是测方法学？
现在 prompt 把整套 recipe 抄给 agent（§2），这是 v1 都高分的根因。要决定：保持处方化（测"能不能可靠跑完固定流程"）， 还是放松成只给目标（测"会不会自己选点 / 选拟合 / 保证一致性"）？可分档：M1 给 recipe、M2/M3 只给目标。
B. T3/T4 已预解平衡结构 → 天生低区分度
T3/T4 直接给 T1 弛豫后的结构，难点（找平衡态）已被解掉，剩下的观测量高度可复现。要决定：是否接受 T3/T4 定位为 "驱动 QE 跑体积 / 应变扫描 + 拟合"的执行类任务（区分度本就低），还是重新设计成更有区分度 （如不给结构、或加扰动结构让 agent 自己先 relax）。
C. GT 单源 → 高一致性有点"循环"
GT 是 weilin self-run（QE 7.5 / V100 / 同一套 GBRV 赝势）产的。agent 用同样的栈跑自然高度一致 —— 一致性高未必证明 agent 强。 要确认：GT 可信度（收敛性看着很好，CaO fit_rms ~5×10⁻⁴ meV/atom）；是否需要第二套独立 GT（不同机器 / 版本）做交叉校验， 或明确"我们就是要测对这套栈的复现"。
D. 0.3 满分比 + 各字段容差是否签字
§1.2 那张表的口径（尤其 V₀ 满分线 0.3% / 零分线 1%）是否符合你对"物理上可接受误差"的判断。 V₀ 在 DFT 里极可复现（~0.1%），1% 归零应该不冤；但请你确认。
E. b0′ 仍用 absolute tol 0.75（≈19%，很松）
B0′ 难收敛 / 敏感，所以保留宽松绝对容差 + 低权重（0.1）。确认这是有意为之，不是漏收。
F. Cij 改特征值比较 — 物理含义确认
特征值比较旋转不变（解决了取向 / 约定误杀），但它只看谱、丢了取向信息：极端情况下，保持谱不变但 C11↔C12 之类错排的矩阵可能漏判（实际很罕见）。要确认：① 旋转不变量（Kelvin moduli）是不是你想要的物理量； ② 要不要额外加一个对称类一致性检查（如 cubic 材料 agent 给的张量是不是 cubic 形）作为补充。
G. 12 道占位题（GT 未 promote）当前已隐藏
这 12 道 GT 是 null / 未 promote，我先 is_stable=False 藏起来了： M1_T4: AlSb / CdSe / CdTe / GaSb / InSb / P_black / Sn_alpha / ZnTe（8）· M2_T3: CaTiO3_tet（1）· M2_T4: CaTiO3_tet / MgAl2O4_spinel / SiO2_alpha_quartz（3）。 要决定：跑 EOS/elastic pipeline 补 GT 再 promote，还是直接从题库删。
H. Web GT 侧 Born 稳定性是 hardcode
§3.2 里 GT 的 Born 稳定性写死 "stable (reference)"，没真算 6×6 本征。bench GT 都是真实收敛材料、必然稳定， 所以目前 OK。确认是否需要真算（要引入一个 6×6 eigensolver）。
